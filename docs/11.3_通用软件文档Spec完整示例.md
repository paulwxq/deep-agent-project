# 通用软件文档 Spec 完整示例

本文档集中给出通用软件文档系统的 spec 示例。

## 1. 通用软件需求规格说明书

```yaml
id: general_software_requirements_specification
name: 需求规格说明书
doc_type: 需求规格说明书
project_type: 通用软件
description: 面向业务方、产品经理、开发与测试团队的需求基线文档。

target_audience:
  - 产品经理
  - 后端研发
  - 测试工程师

template_path: skills/templates/standard_requirements_specification.md

required_sections:
  - 文档目的
  - 背景与范围
  - 术语定义
  - 业务目标
  - 功能需求
  - 非功能需求
  - 范围边界
  - 验收标准

writing_rules:
  - 必须清楚区分“需求”与“实现建议”
  - 功能需求必须可验证
  - 非功能需求必须可量化或可判定
  - 必须写明范围边界与不做事项

review_focus:
  reviewer1:
    - 需求覆盖是否完整
    - 功能需求之间是否冲突
    - 业务目标与需求是否闭环
    - 是否遗漏关键约束
  reviewer2:
    - 是否足以指导后续设计与测试
    - 验收标准是否清楚
    - 术语与结构是否规范统一
    - 是否具备交付质量
```

## 2. 通用软件系统概要设计说明书

```yaml
id: general_software_system_overview_design
name: 系统概要设计说明书
doc_type: 系统概要设计说明书
project_type: 通用软件
description: 面向架构评审与开发实施的系统级概要设计文档。

target_audience:
  - 架构师
  - 后端研发
  - 测试负责人

template_path: skills/templates/standard_system_overview_design.md

required_sections:
  - 项目概述
  - 总体架构设计
  - 核心功能设计
  - 数据架构设计
  - 系统接口设计
  - 非功能性设计

writing_rules:
  - 文档必须能指导后续详细设计与开发实现
  - 不得只写原则，关键模块必须写清职责与边界
  - 必须说明核心技术选型及理由
  - 必须覆盖异常、边界和非功能性约束

review_focus:
  reviewer1:
    - 需求覆盖是否完整
    - 架构是否合理
    - 模块边界是否清晰
    - 是否存在逻辑冲突或关键遗漏
  reviewer2:
    - 是否足以指导实施与评审
    - 接口、异常、约束是否清楚
    - 文档结构与术语是否规范统一
    - 是否达到交付质量
```

## 3. 通用软件系统详细设计说明书

```yaml
id: general_software_system_detailed_design
name: 系统详细设计说明书
doc_type: 系统详细设计说明书
project_type: 通用软件
description: 面向开发实现与测试落地的详细设计文档。

target_audience:
  - 后端研发
  - 测试工程师
  - 技术负责人

template_path: skills/templates/standard_system_detailed_design.md

required_sections:
  - 模块职责设计
  - 关键流程设计
  - 数据模型设计
  - 接口详细定义
  - 异常处理设计
  - 配置与部署约束

writing_rules:
  - 关键接口必须写清入参、出参、异常与约束
  - 数据模型必须写清字段、类型和约束
  - 关键流程必须能直接指导编码
  - 不允许把关键行为留成开放选择题

review_focus:
  reviewer1:
    - 设计是否与需求保持一致
    - 模块职责与流程是否闭环
    - 关键设计是否具体到可实施
    - 是否存在重大逻辑缺口
  reviewer2:
    - 是否足以指导编码与测试
    - 异常、边界、约束是否完整
    - 术语和结构是否一致
    - 是否达到交付标准
```

## 4. 数据仓库/ETL 系统详细设计说明书

```yaml
id: data_warehouse_system_detailed_design
name: 数据仓库系统详细设计说明书
doc_type: 系统详细设计说明书
project_type: 数据仓库/ETL
description: 面向数据开发、数据架构和测试落地的数据仓库/ETL 详细设计文档。

target_audience:
  - 数据架构师
  - 数据开发
  - 数据测试

template_path: skills/templates/data_warehouse_system_detailed_design.md

required_sections:
  - 建设目标与范围
  - 分层设计
  - 数据流与加工链路
  - 表结构与字段口径
  - 调度与依赖
  - 数据质量与异常处理

writing_rules:
  - 必须明确分层、粒度和口径
  - 必须说明来源、加工与落地关系
  - 必须覆盖调度、重跑、失败恢复
  - 必须说明数据质量控制方式

review_focus:
  reviewer1:
    - 分层与口径逻辑是否合理
    - 数据流是否闭环
    - 需求覆盖是否完整
    - 是否存在关键加工遗漏
  reviewer2:
    - 是否足以指导实施与排障
    - 调度、依赖、异常处理是否清楚
    - 结构与术语是否规范统一
    - 是否达到交付质量
```
