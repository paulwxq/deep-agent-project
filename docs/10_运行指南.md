# 运行指南

## 1. 环境准备

**Python 版本**：>= 3.12

**安装依赖**（使用 `uv`）：

```bash
# Windows
set UV_PROJECT_ENVIRONMENT=.venv-win
uv sync

# WSL / Linux
export UV_PROJECT_ENVIRONMENT=.venv-wsl
uv sync
```

如需运行单元测试，还需安装 dev 依赖（包含 `pytest`）：

```bash
uv sync --group dev
```

**配置 API Key**：

```bash
cp .env.sample .env
```

编辑 `.env`，填入实际的 API Key。至少需要配置当前 `config/agents.yaml` 中 Agent 所引用 Provider 对应的 Key。

## 2. 运行方式

两种方式启动程序，效果一致：

**方式 A：通过 `uv run`（推荐，无需手动激活虚拟环境）**

```bash
uv run python main.py -f 需求文件.txt
```

**方式 B：先激活虚拟环境，再用 `python -m`**

```bash
# Windows
.venv-win\Scripts\activate
python -m main -f 需求文件.txt

# WSL / Linux
source .venv-wsl/bin/activate
python -m main -f 需求文件.txt
```

后续示例统一使用方式 A，替换为方式 B 时只需将 `uv run python main.py` 替换为 `python -m main`。

## 3. 基本用法

将需求文件和相关参考文件放入 `./input/` 目录，然后用 `-f` 指定需求文件名：

```bash
# 将需求文件放入 input/ 目录，用纯文件名引用
uv run python main.py -f 需求文件.txt
```

运行结束后，最终文档输出到 `./output/` 目录。

- 如果传了 `-o 文件名.md`，则输出为 `./output/文件名.md`
- 如果没传 `-o`，程序会根据需求内容由 LLM 自动生成文件名（兜底为 `design.md`）

## 4. 命令行参数

| 参数 | 缩写 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `--file` | `-f` | 是 | — | 需求文件名（仅文件名，固定从 `./input/` 读取；若误传路径，自动取 basename 并记录 warning） |
| `--output` | `-o` | 否 | LLM 生成 | 输出文件名（仅文件名，固定写入 `./output/`；若误传路径，自动取 basename 并记录 warning；不传则由 LLM 建议，兜底为 `design.md`） |
| `--config` | `-c` | 否 | `config/agents.yaml` | 配置文件路径 |
| `--max-iterations` | `-m` | 否 | `3`（来自配置文件） | Writer-Reviewer 最大迭代轮数 |
| `--log-level` | `-l` | 否 | `INFO`（来自配置文件） | 控制台日志级别（DEBUG / INFO / WARNING / ERROR） |

## 5. 常用示例

```bash
# 基本用法
uv run python main.py -f 需求文件.txt

# 指定输出文件名
uv run python main.py -f 需求文件.txt -o 设计方案.md

# 增加迭代轮数到 5 轮
uv run python main.py -f 需求文件.txt -m 5

# 控制台显示 DEBUG 级别日志（排查问题时使用）
uv run python main.py -f 需求文件.txt -l DEBUG

# 使用自定义配置文件
uv run python main.py -f 需求文件.txt -c my-config.yaml

# 组合使用
uv run python main.py -f my-requirement.md -o report.md -m 4 -l INFO
```

## 6. 日志

| 通道 | 级别 | 位置 |
|------|------|------|
| 控制台 | 由 `--log-level` 或配置文件 `global.log_level` 控制 | stderr |
| 文件 | 始终 DEBUG（完整记录模型输出、推理过程、工具调用） | `logs/agent.log` |

日志文件自动轮转，单文件上限 10 MB，保留最近 5 个备份。

## 7. 目录结构

```
deep-agent-project/
├── input/              # 常驻 | 用户输入目录（只读，程序不会修改或清理）
│   ├── 需求文件.txt    #         主需求文件（由用户预先放置）
│   └── *.sas, *.csv   #         参考文件（Agent 按需阅读）
├── output/             # 常驻 | 最终产物输出目录（路径固定，不可通过参数修改）
├── config/
│   └── agents.yaml     # 常驻 | LLM Provider、Agent 模型、工具配置
├── drafts/             # 运行时 | Agent 工作区（每次运行前自动备份并清空，git 忽略）
│   ├── design.md       #         Writer 输出的设计文档草稿
│   ├── review-verdict.json  #    Reviewer 的结构化结论
│   └── output-filename.txt  #    Orchestrator 建议的输出文件名
├── logs/               # 运行时 | 日志文件（git 忽略）
│   └── agent.log
├── tests/              # 单元测试（pytest）
├── src/                # 源代码
├── skills/             # Agent 技能文件
├── docs/               # 项目文档
├── .env                # API Key（git 忽略，从 .env.sample 复制）
└── main.py             # 入口
```

**常驻目录**（纳入 git）：`input/`、`output/`、`config/`、`src/`、`skills/`、`docs/`

**运行时目录**（git 忽略）：`drafts/`、`logs/`。其中 `drafts/` 每次运行前会自动备份为 `drafts_YYYYMMDD_HHMMSS/`。

## 8. 运行输出

运行结束后，控制台会打印两部分信息（通过 logger 输出）：

1. **执行统计**（程序精确计数）：

```
[INFO] [system] 执行统计: 共 5 次子代理委派 (reviewerx2, writerx3)
```

2. **迭代摘要**（由 Orchestrator 生成）：包含迭代轮数、最终审核结论、关键改进点。

## 9. 输出文件名的确定规则

程序通过三级降级确定最终的输出文件名，所有来源都经过 `sanitize_filename()` 统一清洗：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1 | `-o` 参数 | 用户显式指定的文件名（自动取 basename，清洗后使用） |
| 2 | `drafts/output-filename.txt` | Orchestrator 在迭代结束后根据需求内容建议的文件名 |
| 3 | `design.md` | 兜底默认值（上述两个来源都不可用时使用） |

`sanitize_filename()` 的清洗规则：strip -> 只取第一行 -> 取 basename -> 过滤非法字符 -> 去除首尾点和空格 -> Windows 保留名兜底 -> 补 `.md` 后缀。

## 10. 单元测试

项目在 `tests/` 目录下维护单元测试，覆盖 `sanitize_filename()` 和 `load_config()` 等核心逻辑。

**运行全部测试**：

```bash
# 安装 dev 依赖（仅首次）
uv sync --group dev

# 运行全部测试（详细输出）
uv run pytest -v
```

**运行单个测试文件**：

```bash
uv run pytest tests/test_sanitize_filename.py -v
uv run pytest tests/test_config_loader.py -v
```

预期输出（全部通过）：

```
tests/test_sanitize_filename.py ..................................................   [ 73%]
tests/test_config_loader.py ..................                                       [100%]

69 passed in 0.11s
```
