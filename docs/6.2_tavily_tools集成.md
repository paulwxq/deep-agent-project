# 6.2 Tavily Web 搜索工具集成

## 0. 背景与定位

| 维度 | Context7（MCP） | Tavily（Tool） |
|------|----------------|----------------|
| 协议 | Model Context Protocol，HTTP 远程服务 | LangChain 原生 `BaseTool`，同步函数 |
| 用途 | 查阅第三方库官方文档和 API 规范 | 搜索互联网，获取最新技术资讯、社区讨论、行业数据 |
| 网络 | 需要访问 `mcp.context7.com` | 需要访问 `api.tavily.com` |
| 异步 | 必须异步调用（`async`） | 同步调用，无事件循环限制 |
| 加载时机 | `_async_main()` 启动时异步加载 | `create_orchestrator_agent()` 同步创建时加载 |

Tavily 擅长回答"现在发生了什么"类问题（最新版本号、近期 Bug 报告、社区最佳实践），是对 Context7 文档检索能力的互补。

## 3. 配置文件

在 `config/agents.yaml` 的 `tools` 节点下启用 Tavily：

```yaml
tools:
  tavily:
    enabled: true           # 改为 true 启用
    api_key_env: "TAVILY_API_KEY"   # 环境变量名，与 .env 对应
    max_results: 5          # 每次搜索返回的最大结果数
```

**字段说明：**

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | `false` | 总开关，`false` 时不加载工具、不校验 Key |
| `api_key_env` | string | `"TAVILY_API_KEY"` | 存放 Key 的环境变量名，支持自定义 |
| `max_results` | int | `5` | 搜索结果条数上限（建议 3–10） |

## 4. 实现路径（代码已完成）

Tavily 集成涉及四个文件，**代码均已实现，无需修改**：

### 4.1 工具实现：`src/tools/web_search.py`

```python
def create_web_search_tool(
    max_results: int = 5,
    api_key_env: str = "TAVILY_API_KEY",
):
    """创建 Tavily 网络搜索工具函数。"""
    from tavily import TavilyClient

    api_key = os.environ.get(api_key_env, "")
    if not api_key:
        raise KeyError(f"环境变量 '{api_key_env}' 未设置，请在 .env 文件中配置")

    client = TavilyClient(api_key=api_key)

    def internet_search(
        query: str,
        num_results: int = max_results,
        topic: Literal["general", "news", "finance"] = "general",
    ) -> str:
        """搜索互联网获取最新信息。用于查找技术文档、最佳实践、API 参考等。"""
        results = client.search(query, max_results=num_results, topic=topic)
        return json.dumps(results, ensure_ascii=False)

    return internet_search
```

**设计要点：**
- `TavilyClient` 在工厂函数内延迟导入，`tavily_enabled=false` 时不导入、不产生依赖
- `internet_search` 的 docstring 即 LangChain 的工具描述，模型根据此决定是否调用
- `topic` 参数支持 `general` / `news` / `finance`，方便模型针对性搜索

### 4.2 配置解析：`src/config_loader.py`

```python
@dataclass
class ToolsConfig:
    tavily_enabled: bool = False
    tavily_api_key_env: str = "TAVILY_API_KEY"
    tavily_max_results: int = 5
    context7: Context7Config = field(default_factory=Context7Config)
```

`load_config()` 从 `tools.tavily` 节点读取三个字段，填充到 `ToolsConfig`。

`validate_env_vars()` 只在 `tavily_enabled=true` 时检查 `TAVILY_API_KEY`：

```python
if config.tools.tavily_enabled:
    if not os.environ.get(config.tools.tavily_api_key_env):
        missing.append(config.tools.tavily_api_key_env)
```

### 4.3 工具注册：`src/agent_factory.py`

```python
# create_orchestrator_agent() 中
tools: list = []
if config.tools.tavily_enabled:
    from src.tools.web_search import create_web_search_tool
    tools.append(create_web_search_tool(
        max_results=config.tools.tavily_max_results,
        api_key_env=config.tools.tavily_api_key_env,
    ))

# Writer 工具（含 tavily + context7 + ask_user）
writer_tools = list(tools) + list(context7_tools)
if config.hil_clarify:
    writer_tools.append(ask_user)

# Reviewer 工具（含 tavily + context7）
reviewer_tools = list(tools) + list(context7_tools)
```

**关键决策：Tavily 同时注册给 Writer 和 Reviewer**

- **Writer** 用 Tavily 进行研究型搜索：查阅最新技术方案、行业标准、开源社区讨论
- **Reviewer** 用 Tavily 进行验证型搜索：核查设计文档中引用的数据、版本号、工具是否仍然有效

Orchestrator 不持有 Tavily 工具——它的职责是任务调度，不做内容研究。

### 4.4 环境变量校验：`main.py` 启动流程

```
main()
  └─ load_config()              # 读取 YAML
  └─ validate_env_vars(config)  # tavily_enabled=true → 检查 TAVILY_API_KEY
  └─ 若缺失 → logger.error + sys.exit(1)  # 启动时 fail-fast
  └─ asyncio.run(_async_main()) # 正常启动
       └─ create_orchestrator_agent(config, ..., context7_tools)
            └─ create_web_search_tool(...)  # 同步创建，无需 await
```

Tavily 工具是同步函数，在 `create_orchestrator_agent()` 中直接创建，不需要 `async`，与 MCP 的异步加载路径完全独立。

---

## 5. 工具调用流（运行时）

```
Orchestrator 委派任务给 Writer
  └─ Writer 分析需求，判断需要最新信息
  └─ Writer 调用 internet_search(query="langchain tool calling 2025 best practices")
  └─ Tavily 返回结构化搜索结果（含标题、URL、摘要）
  └─ Writer 将搜索结果整合进设计文档

Orchestrator 委派审核给 Reviewer
  └─ Reviewer 发现设计中某版本号可疑
  └─ Reviewer 调用 internet_search(query="langgraph 1.1 release notes")
  └─ Reviewer 根据搜索结果给出 REVISE 或 ACCEPT 结论
```

## 7. 故障排除

### 7.1 启动时报 "以下环境变量未设置: TAVILY_API_KEY"

原因：`tavily.enabled=true` 但 `.env` 中未填 Key，或 `.env` 文件未被加载。

解决：
1. 确认 `.env` 文件在项目根目录（与 `main.py` 同级）
2. 确认 `TAVILY_API_KEY=tvly-xxx` 行不含多余空格或引号错误

### 7.2 网络访问失败（代理环境）

Tavily 客户端底层使用 `requests` 库，会自动读取系统代理环境变量：

```bash
# 在 .env 或 shell 中设置
HTTP_PROXY=http://127.0.0.1:7890
HTTPS_PROXY=http://127.0.0.1:7890
```

> 注意：Context7 MCP 使用 `httpx`，需要额外安装 `httpx[socks]` 才能走 SOCKS 代理（详见 6.1）；Tavily 使用 `requests`，直接设置 HTTP_PROXY 即可。

### 7.3 搜索结果质量差

调整 `max_results`（建议 3–8）和 `topic` 参数：

```yaml
tools:
  tavily:
    enabled: true
    max_results: 8   # 增大结果数以获取更全面的信息
```

模型调用时可通过 `topic="news"` 参数聚焦最新资讯。

### 7.4 Token 消耗过多

Tavily 返回的是摘要而非完整网页，通常每条结果约 200–300 Token。`max_results=5` 时单次搜索约消耗 1000–1500 Token。

若 Context Window 紧张，建议将 `max_results` 降至 3。

## 8. 与 Context7 的协同使用

两个工具可同时启用，互不冲突：

```yaml
tools:
  tavily:
    enabled: true
    max_results: 5

  context7:
    enabled: true
    api_key_env: "CONTEXT7_API_KEY"
```

模型会根据任务类型自主选择：

| 查询类型 | 推荐工具 | 原因 |
|----------|---------|------|
| "LangGraph 的 `astream()` 参数怎么用？" | Context7 | 官方文档，精准且结构化 |
| "2025 年最流行的 RAG 框架有哪些？" | Tavily | 需要互联网最新资讯 |
| "langgraph 1.1.3 的 changelog" | Tavily | 版本发布信息在 GitHub/博客，非文档 |
| "`MemorySaver` 的线程安全说明" | Context7 | API 规范，文档更权威 |

在 System Prompt 中两个工具的使用说明并列呈现，模型会依据 docstring 描述自动路由（无需额外提示）。

**`topic` 参数使用技巧：**

`internet_search` 暴露了 `topic` 参数给模型，但模型默认倾向于使用 `general`。在系统提示词中加入以下引导语，可以让模型更主动地利用 `news` 类型：

```
搜索最新技术动态、版本发布信息或近期 Bug 报告时，
请在调用 internet_search 时指定 topic="news"，以获得更及时的结果。
```

该提示目前**未内置在 Writer/Reviewer 的 System Prompt 中**。若实际使用发现模型对版本信息或最新资讯的搜索质量不佳，可在 `src/prompts/writer_prompt.py` 和 `reviewer_prompt.py` 的 `tavily_section`（待实现，参考 `context7_section` 模式）中补充此引导。
