### 需求说明：
1.该需求要使用langchain-ai的deepagents模块的sdk来开发，所有设计都要符合deepagents的文档或最佳实践。
2.至少包含两个agent/subagent，一个负责按照业务需求编写技术设计文档(md格式)，一个负责基于业务需求审核技术设计文档，如果不符合要求，就打回去重写，直到合格。
3.你可以控制两个subagent交互的轮数，如果有需要可以添加一个agent/subagent用来作为仲裁或任务编排
4.根据项目需要，你可以提出来为这个deepagents项目配置skill和web search tools（Tivily等）.如果deepagents模块支持skil，我很乐意提供。
5.writer agent/subagent 能根据我的需求写出能够落地的技术设计文档。
6.reviewer agent/subagent 负责审核技术设计文档，是否足够指导代码落地，是否还有歧义，也不要让coder做出选择，writer agent要给出最佳建议。对于设计文档中的选择或可讨论的内容，reviewer可以一起参与给出建议。对于设计文档中不合理的内容，reviewer agent也要给出建议。但不能要求writer一定要接受这个建议，writer可以有自己的想法，只要输出的设计文档符合要求，reviewer就要放行。
7.项目要支持多种llm，比如qwen/minimax/glm/openrouter等，每个agent/subagent 调用的llm最好做成可配置的，能够在yaml文件中配置llm的provider,model_id,以及它们各自支持的参数。
8.项目要使用logger的模块把日志输出到log日志文件中，要可以区分logger.debug和logger.info，并且可以根据大小自动滚动。多个agent/subagent之间的交流要写入到日志中，让我们知道agent直接的对话。
9.当前项目使用uv 或 python 运行代码时，可以通过命令行传入简单的需求，也可以指定需求的md文档的名字，默认路径就在项目根目录下。
10.请不要弄混Deep Agents CLI 与 deepagents sdk,Deep Agents CLI是无法自定义的。


### 技术栈：
- python >=3.12
- deepagents >=0.4.11
- langchain >=1.2.20
- langgraph >=1.1.3
- 使用uv管理项目依赖，当前项目将会有两个python虚拟环境，一个是.venv-wsl在wsl下，一个是.venv-win在windows下。
