# 6.1 Context7 MCP 集成实施指南

## 概述

本文档描述如何将 **Context7**（技术文档查询服务）以 **HTTP MCP 方式**集成到当前 Writer-Reviewer Agent 系统，使 Writer 和 Reviewer 均可通过工具调用查询最新的第三方库文档。

**设计要点**：
- 使用 Context7 的**远程 HTTP 端点** (`https://mcp.context7.com/mcp`)，无需本地安装 Node.js 或任何 CLI 工具
- 通过 `langchain-mcp-adapters` 将 MCP 工具转为 LangChain `BaseTool`，与现有 `create_deep_agent()` 无缝对接
- MCP 工具是异步协程，必须全链路使用 `agent.ainvoke()` / `agent.astream()`（不能用同步 `agent.invoke()`）
- Writer 和 Reviewer 获得相同工具，保证信息对等

**涉及修改的文件**：

| 文件 | 类型 | 说明 |
|------|------|------|
| `pyproject.toml` | 修改 | 新增两个依赖包 |
| `.env` / `.env.sample` | 修改 | 新增 `CONTEXT7_API_KEY` |
| `config/agents.yaml` | 修改 | 新增 `tools.context7` 配置节 |
| `src/config_loader.py` | 修改 | 新增 `Context7Config` 数据类，更新 `ToolsConfig` 和 `validate_env_vars` |
| `src/tools/context7_mcp.py` | **新建** | 异步加载 Context7 MCP 工具 |
| `src/agent_factory.py` | 修改 | 新增 `context7_tools` 参数，注入到 Writer/Reviewer |
| `main.py` | 修改 | 重构为 `asyncio.run(_async_main())`，HIL 路径改为 async |
| `src/prompts/writer_prompt.py` | 修改 | 新增 `context7_enabled` 参数，注入工具使用指引 |
| `src/prompts/reviewer_prompt.py` | 修改 | 同上 |

## 5. `src/tools/context7_mcp.py`（新建）

```python
"""Context7 MCP 工具加载器。

使用 langchain-mcp-adapters 通过 HTTP 连接 Context7 远程 MCP 服务，
将工具转为 LangChain BaseTool 对象，可直接传给 create_deep_agent()。

无需本地安装 Node.js 或 Context7 CLI——全程走网络请求。
"""

from __future__ import annotations

import logging
import os

_log = logging.getLogger("deep_agent_project")


async def load_context7_tools(
    api_key_env: str = "CONTEXT7_API_KEY",
    url: str = "https://mcp.context7.com/mcp",
) -> list:
    """异步加载 Context7 MCP 工具列表。

    返回的工具列表可直接传入 create_deep_agent(tools=...)。
    工具会持有对 MCP 会话的引用，调用方须确保在使用工具期间
    MultiServerMCPClient 对象保持存活（不被垃圾回收）。

    Args:
        api_key_env: 存放 API Key 的环境变量名。
        url: Context7 MCP 端点 URL。

    Returns:
        BaseTool 列表；若加载失败则返回空列表并记录 warning。
    """
    from langchain_mcp_adapters.client import MultiServerMCPClient

    api_key = os.environ.get(api_key_env, "")
    if not api_key:
        _log.warning("Context7 MCP: 环境变量 %s 未设置，跳过加载", api_key_env)
        return []

    try:
        client = MultiServerMCPClient({
            "context7": {
                "transport": "http",
                "url": url,
                "headers": {"CONTEXT7_API_KEY": api_key},
            }
        })
        tools = await client.get_tools()
        _log.info("Context7 MCP: 已加载 %d 个工具 → %s", len(tools), [t.name for t in tools])
        return tools
    except Exception as exc:
        _log.warning("Context7 MCP 加载失败（跳过，不影响主流程）: %s", exc)
        return []
```

> **生命周期说明**：`MultiServerMCPClient` 建立 HTTP 长连接后，工具对象持有对该连接的引用。只要 `client` 变量留在调用栈的作用域内，连接就保持活跃。在 `_async_main()` 中将 `client` 和 `tools` 都赋值给局部变量即可——整个 agent 执行期间两者都存活。

## 9. `main.py` 异步重构

这是改动最大的文件。核心原因：Context7 MCP 工具是异步协程，在 `asyncio.run()` 内部调用同步 `agent.invoke()` 时会引发 `"This event loop is already running"` 错误，必须全链路使用 async。

### 9.1 整体结构

```python
import asyncio

def main() -> None:
    # 前段：argparse、日志初始化、配置加载、文件校验、drafts 备份
    # ── 这部分保持同步 ──────────────────────────────────────────────
    args = ...
    config = ...
    ...
    _backup_drafts_contents(drafts_dir)

    # 后段：Agent 创建和执行 → 改为 async
    try:
        asyncio.run(_async_main(config, requirement_filename, output_filename_from_arg, logger))
    except KeyboardInterrupt:
        # ⚠️  asyncio.run() 在 Python 3.12 下会在 Ctrl+C 时取消主 task、
        # 清理 MCP AsyncExitStack，然后重新抛出 KeyboardInterrupt。
        # 这里捕获后静默退出，避免用户看到难看的 CancelledError traceback。
        #
        # ❌ 不要用 signal.signal(SIGINT, default_int_handler)：
        #    那会覆盖 asyncio 自己的 SIGINT handler，破坏 graceful shutdown，
        #    导致 MCP 连接来不及关闭。
        print()   # 补一个换行，避免 "^C" 与下一行提示符粘在一起
        sys.exit(130)   # 130 = 128 + SIGINT(2)，Unix 标准 Ctrl+C 退出码
    except SystemExit:
        raise
    except Exception:
        logger.exception("Agent 执行失败")
        sys.exit(1)


async def _async_main(config, requirement_filename, output_filename_from_arg, logger) -> None:
    """Agent 创建与执行（全链路 async，支持 MCP 工具调用）。"""
    from src.agent_factory import create_orchestrator_agent
    from src.tools.context7_mcp import load_context7_tools

    # 1. 加载 Context7 MCP 工具（如未启用，返回空列表）
    context7_tools = []
    if config.tools.context7.enabled:
        context7_tools = await load_context7_tools(
            api_key_env=config.tools.context7.api_key_env,
            url=config.tools.context7.url,
        )
        # ⚠️ 注意：context7_tools 中的工具持有对 MCP 会话的引用。
        # 必须保证 context7_tools 在整个 agent 执行期间不被 GC 回收。
        # 本函数中 context7_tools 是局部变量，生命周期覆盖后续所有代码，满足此要求。

    # 2. 创建 Agent（同步函数，仅构建图，不执行）
    agent, orch_middleware = create_orchestrator_agent(
        config, requirement_filename, context7_tools=context7_tools
    )
    print_system("Agent 创建完成，开始执行…")

    initial_messages = [
        {
            "role": "user",
            "content": (
                f"请根据需求编写技术设计文档。"
                f"需求文件在 /input/{requirement_filename}，"
                f"同目录下的其他文件为参考文件，请按需阅读。"
            ),
        }
    ]
    thread_config = {"configurable": {"thread_id": str(uuid.uuid4())}}

    # 3. 执行 Agent（必须使用 async 接口）
    interactive = config.hil_clarify or config.hil_confirm
    if interactive:
        result = await _run_with_hil_async(agent, initial_messages, thread_config, logger)
    else:
        result = await agent.ainvoke(   # ← ainvoke 而非 invoke
            {"messages": initial_messages}, config=thread_config
        )

    # 4. 后处理（输出文件名确定、文件复制、摘要输出）
    # ── 与原 main() 中第 10-13 步完全相同 ──────────────────────────
    ...
    print_final_summary(output_filename, orch_middleware.task_counts, final_message)
```

### 9.2 `_run_with_hil_async()` 完整实现

原 `_run_with_hil()` 使用 `agent.stream()`（同步生成器），在 async 上下文中需改为 `agent.astream()`：

```python
async def _run_with_hil_async(agent, initial_messages: list, thread_config: dict, logger) -> dict:
    """HIL 交互路径的异步版本。

    原 _run_with_hil() 使用 agent.stream()（同步生成器）；
    改为 agent.astream()（异步生成器），使 MCP 工具调用可正确 await。

    注意：input() 在 async 函数中调用仍然安全——Python 的 input() 是
    阻塞式 OS syscall，asyncio 允许其在协程中使用（不推荐用于高并发
    服务，但对本项目的交互式 CLI 场景无影响）。
    """
    import re as _re
    from langgraph.types import Command

    _Q_PATTERN = _re.compile(r'^Q(\d+)[:.：、]\s*(.+)$')
    _EXIT_CMDS = {"quit", "exit"}

    payload = {"messages": initial_messages}
    result = None

    while True:
        interrupted = False
        # ↓ astream 替换 stream
        async for event in agent.astream(payload, config=thread_config):
            interrupts = event.get("__interrupt__")
            if not interrupts:
                continue

            interrupted = True
            interrupt_value = interrupts[0].value

            # ── 第一类：需求澄清 ──────────────────────────────────────
            if "questions" in interrupt_value:
                questions = interrupt_value.get("questions", "（Agent 未提供具体问题）")
                q_matches = [
                    (int(m.group(1)), line.strip())
                    for line in questions.splitlines()
                    if (m := _Q_PATTERN.match(line.strip()))
                ]
                actual_nums = [num for num, _ in q_matches]
                expected_nums = list(range(1, len(q_matches) + 1))
                protocol_valid = (
                    2 <= len(q_matches) <= 3
                    and actual_nums == expected_nums
                )

                print_ask_user(questions)

                if protocol_valid:
                    console.print("  [dim]（请逐条回答，每条输入完成后按回车；输入 quit 终止）[/dim]")
                    answers = []
                    for num, _ in q_matches:
                        print(f"A{num}：", end="", flush=True)
                        ans = _read_console_input(logger=logger).strip()
                        if ans.lower() in _EXIT_CMDS:
                            raise SystemExit(0)
                        while not ans:
                            ans = _read_console_input(logger=logger).strip()
                            if ans.lower() in _EXIT_CMDS:
                                raise SystemExit(0)
                        answers.append(f"A{num}：{ans}")
                    user_answer = "\n".join(answers)
                else:
                    if len(q_matches) > 3:
                        logger.warning("问题数量超过上限 3 个，降级为自由文本回答")
                    lines: list[str] = []
                    while True:
                        line = _read_console_input(logger=logger)
                        if line.lower() in _EXIT_CMDS:
                            raise SystemExit(0)
                        if line == "" and lines:
                            break
                        if line:
                            lines.append(line)
                    user_answer = "\n".join(lines)

                payload = Command(resume=user_answer)

            # ── 第二类：超限确认 ──────────────────────────────────────
            elif "status" in interrupt_value:
                status = interrupt_value.get("status", "迭代已达上限")
                print_confirm_continue(status)
                _YES_INPUTS = {"yes", "y", "继续", "是"}
                _NO_INPUTS = {"no", "n", "否"}
                while True:
                    choice = _read_console_input(logger=logger).strip().lower()
                    if choice in _EXIT_CMDS:
                        raise SystemExit(0)
                    if choice in _YES_INPUTS or choice in _NO_INPUTS:
                        break
                    print("请输入 yes 或 no（输入 quit 退出）：", end="", flush=True)
                user_answer = "yes" if choice in _YES_INPUTS else "no"
                print_system("用户授权继续" if user_answer == "yes" else "用户选择结束迭代")
                payload = Command(resume=user_answer)

            # ── 未知中断：安全兜底 ────────────────────────────────────
            else:
                logger.warning("收到未知类型的 HIL 中断，自动以空字符串恢复")
                payload = Command(resume="")

            break  # 退出本轮 astream，用新 payload 重新进入 while

        if not interrupted:
            result = agent.get_state(config=thread_config).values
            break

    return result
```

> **与原 `_run_with_hil()` 的差异**：只有三处改动：
> 1. `def` → `async def`
> 2. `agent.stream(...)` → `agent.astream(...)`（以及 `for` → `async for`）
> 3. 接受 `logger` 参数（原来通过模块级 `import logging` 获取）

import asyncio

## 附录：Context7 工具使用方式

当前正式工具（v2.0.0+ / 2025-12 起）：

| 工具名 | 作用 | 关键参数 |
|--------|------|---------|
| `resolve-library-id` | 将库名解析为 Context7 标准 ID | `libraryName` |
| `query-docs` | 按问题查询文档片段 | `libraryId`, `query` |

> `get-library-docs` 已在 v2.0.0 弃用，参数也从 `context7CompatibleLibraryID + topic` 改为 `libraryId + query`。

两步固定流程示例：

```
# 第 1 步：解析库 ID
resolve-library-id(libraryName="fastapi")
→ 返回: "/tiangolo/fastapi"

# 第 2 步：查询文档
query-docs(libraryId="/tiangolo/fastapi", query="dependency injection")
→ 返回: FastAPI 依赖注入的最新文档片段
```

**不要跳过第 1 步**：直接猜测库 ID 填入 `query-docs` 容易出错，必须先通过 `resolve-library-id` 获取标准 ID。

prompt 中注入的工具名来自运行时动态加载结果，不依赖本附录中的硬编码名称。
