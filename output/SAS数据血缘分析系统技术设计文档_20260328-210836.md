# SAS数据血缘分析系统技术设计文档

## 1. 项目概述

### 1.1 项目背景
本项目旨在开发一个Python应用模块，用于分析SAS代码和相关数据集，生成数据血缘分析对照表。该系统将帮助数据工程师和分析师理解数据从源表到目标表的转换过程，追踪字段级别的数据血缘关系。

### 1.2 核心需求
- **输入数据类型**：
  - SAS代码文件（.sas）
  - 源数据：大部分定义在mainframe上（通过def_*.sas文件定义），小部分为CSV或Excel文件
  - 中间临时文件：SAS文件（.sas7bdat后缀）
  - 输出文件：CSV格式
  
- **输出要求**：
  - 数据血缘对照表，包含：output_table, output_column, original_table, original_column, formula, script_file
  - 从输出表作为分析入口，反向追踪每个结果表字段对应的源表字段
  - 支持复杂的字段转换逻辑（重命名、计算表达式、多表合并等）
  - 输出格式与提供的Output_sas_20260318.csv示例保持一致
  - 支持多源字段（original_table和original_column使用分号分隔多个值）

### 1.3 技术约束
- Python 3.12
- 可配置的LLM Provider支持（OpenAI、Anthropic等）
- 使用LangGraph进行智能分析工作流编排
- 必须处理实际SAS代码中的复杂场景（如中文Excel工件、字段重命名、跨表计算等）
- 支持mainframe数据定义文件（def_*.sas）解析
- 支持.sas7bdat中间文件读取和处理

## 2. 系统架构设计

### 2.1 整体架构
系统采用分层架构设计，包含以下核心组件：

```
┌─────────────────────────────────────────────────────────────┐
│                    SAS数据血缘分析系统                      │
├───────────────────┬───────────────────┬─────────────────────┤
│   输入处理层       │   分析引擎层       │   输出管理层         │
├───────────────────┼───────────────────┼─────────────────────┤
│ • 文件发现器       │ • SAS解析器        │ • 血缘图构建器       │
│ • 文件读取器       │ • LLM增强分析器    │ • CSV导出器         │
│ • 配置管理器       │ • 血缘追踪器       │ • API接口           │
└───────────────────┴───────────────────┴─────────────────────┘
```

### 2.2 数据流设计
1. **文件发现阶段**：扫描指定目录，识别SAS代码文件、CSV文件和SAS数据集文件
2. **代码解析阶段**：解析SAS代码，提取数据处理逻辑、表映射关系和字段转换规则
3. **血缘分析阶段**：结合LLM智能分析，构建完整的数据血缘图谱
4. **结果输出阶段**：生成标准化的数据血缘对照表

### 2.3 技术选型
- **核心框架**：LangGraph（状态化工作流引擎）
- **LLM集成**：LangChain的可配置LLM Provider（支持OpenAI、Anthropic等）
- **文件处理**：内置文件系统操作工具（read_file, write_file, ls, grep等）
- **SAS文件处理**：sas7bdat库（用于读取.sas7bdat中间文件）
- **上下文管理**：智能分块和摘要处理，避免LLM上下文限制

### 3.2 SAS代码解析模块

#### 3.2.1 SAS语法解析器 (SASParser)
- **功能**：解析SAS代码的关键结构
- **关键语句识别**：
  - `PROC IMPORT`：识别源数据文件和目标数据集
  - `DATA` 语句：识别数据集创建和转换逻辑
  - `SET` 语句：识别输入数据集和字段重命名
  - `MERGE` 语句：识别多表合并操作
  - `KEEP` 语句：识别保留的字段列表

#### 3.2.2 Mainframe定义文件解析器 (MainframeDefParser)
- **功能**：专门解析def_*.sas格式的mainframe数据定义文件
- **解析内容**：
  - **INFILE语句**：提取文件路径、记录格式（recfm=F）和记录长度（lrecl=657）
  - **INPUT语句**：解析字段位置（@1, @3等）、字段名和字段格式
  - **字段格式映射**：
    - `PK2.0`, `PK3.0`：packed decimal格式，表示数值字段
    - `S370FPD6.0`, `S370FPD7.0`：IBM S/370 floating point decimal，表示数值字段
    - `$EBCDIC3.`：EBCDIC编码的字符字段
- **血缘关联**：将解析出的字段定义与实际数据血缘关联，作为源字段的元数据来源
- **示例处理**：对于`def_Churn.sas`中的`@1 CustomerId PK2.0`，解析为字段名"CustomerId"，位置1，数值类型
  - `PROC EXPORT`：识别输出文件路径

#### 3.2.2 字段映射提取器 (FieldMapper)
- **功能**：从SAS代码中提取字段级别的映射关系
- **映射类型**：
  - **直通映射**：字段直接传递（如 `CustomerId` → `CustomerId`）
  - **重命名映射**：字段重命名（如 `CreditScore` → `credit_scr`）
  - **计算映射**：派生字段计算（如 `bal_per_prod = Balance / num_prod`）
  - **条件映射**：条件表达式（如 `senior_flag = (Age >= 60)`）

```python
def extract_lineage(state: AnalysisState) -> dict:

    # 使用LLM或规则引擎提取血缘
```

#### 3.3.4 错误处理机制
- **JSON格式错误**：当LLM返回非JSON格式时，尝试使用正则表达式提取关键信息作为降级方案
- **重试机制**：最多重试3次，每次调整提示词或减少上下文长度
- **人工审核标记**：标记分析失败的文件，供人工审核

#### 3.3.5 上下文分块策略
针对大型SAS代码库，采用智能分块策略：
- **按文件分块**：每个SAS文件作为一个独立分析单元
- **按逻辑分块**：将大型SAS文件按DATA步、PROC步分割（使用正则表达式识别步骤边界）
- **依赖感知分块**：确保相关联的代码片段在同一上下文中处理
- **增量分析**：只重新分析修改过的代码部分

### 3.4 血缘追踪与图谱构建模块

#### 3.4.1 血缘追踪器 (LineageTracker)
- **功能**：从输出表反向追踪到源表
- **追踪算法**：
  1. 识别所有输出CSV文件
  2. 对每个输出文件，找到生成它的SAS代码
  3. 解析SAS代码中的输入数据集（包括中间.sas7bdat文件）
  4. 递归追踪输入数据集的来源，直到到达源数据（mainframe定义文件或源CSV/Excel）
  5. 处理MERGE操作中的多源字段追踪
  6. 检测并避免循环依赖

#### 3.4.2 血缘图构建器 (LineageGraphBuilder)
- **功能**：构建完整的数据血缘图谱
- **图结构**：
  - **节点**：表/文件 + 字段
  - **边**：字段映射关系（包含转换公式）
  - **元数据**：SAS脚本文件路径、行号等
- **存储方式**：内存中的字典结构，键为"table.field"，值为血缘记录列表
- **持久化**：通过CSV导出器保存到文件系统

## 4. 数据血缘分析算法

### 4.1 基于规则的静态分析
#### 4.1.1 SAS代码模式匹配
- **正则表达式模式**：
  ```python
  # DATA语句模式
  data_pattern = r'DATA\s+(\w+);'
  # SET语句重命名模式  
  rename_pattern = r'SET\s+\w+\s*\(RENAME=\(([^)]+)\)\);'
  # 字段赋值模式
  assignment_pattern = r'(\w+)\s*=\s*([^;]+);'
  ```

#### 4.1.2 字段依赖分析
- **输入字段识别**：从计算表达式中提取依赖字段
- **输出字段识别**：从KEEP语句或DATA步中识别输出字段
- **依赖图构建**：建立字段间的依赖关系

### 4.2 LLM增强的语义分析
#### 4.2.1 复杂表达式解析
对于复杂的SAS表达式，使用LLM进行语义理解：
- **输入**：SAS表达式 + 上下文信息
- **输出**：结构化的字段依赖关系
- **示例**：
  ```
  输入: risk_score = credit_scr - (dti * 10)
  输出: {"output_field": "risk_score", "input_fields": ["credit_scr", "dti"], "formula": "credit_scr - (dti * 10)"}
  ```

#### 4.2.2 多表合并分析
对于MERGE操作，使用LLM理解跨表字段关系：
- **输入**：MERGE语句 + 相关DATA步
- **输出**：跨表字段映射关系
- **处理逻辑**：
  1. 识别MERGE的输入表
  2. 分析每个输入表的字段来源
  3. 构建跨表字段依赖关系

### 4.3 反向血缘追踪算法
#### 4.3.1 算法流程
```
function traceLineage(outputTable, outputField, visitedNodes = set()):
    1. 检查visitedNodes，如果(outputTable, outputField)已存在，返回空（避免循环）
    2. 将(outputTable, outputField)加入visitedNodes
    3. 找到生成outputTable的SAS脚本
    4. 在脚本中定位outputField的定义
    5. 如果outputField来自中间文件(.sas7bdat)：
       a. 读取中间文件的元数据获取字段信息
       b. 找到生成该中间文件的上游SAS脚本
       c. 递归追踪上游脚本中的对应字段
    6. 如果outputField来自MERGE操作：
       a. 识别所有输入表（可能来自多个源）
       b. 对每个输入表中的对应字段，递归调用traceLineage
       c. 合并所有源路径（使用分号分隔多个源）
    7. 如果是直通字段，递归追踪源字段
    8. 如果是计算字段，解析表达式中的输入字段，对每个输入字段递归调用traceLineage
    9. 返回完整的血缘路径
```

#### 4.3.2 中间文件处理
- **.sas7bdat文件处理**：
  - 使用`sas7bdat`库读取中间文件获取字段元数据
  - 建立中间文件与生成它的SAS代码的关联关系
  - 在反向追踪时，将中间文件作为临时节点，继续向上游追踪

#### 4.3.3 多源字段追踪
- **MERGE操作处理**：
  - 识别MERGE语句中的所有输入数据集
  - 对于目标字段，分析其在各个输入数据集中的来源
  - 支持多源字段的血缘追踪（如示例中的`risk_score`字段来自`credit_scr`和`dti`两个不同源）

#### 4.3.4 递归终止条件
- **终止条件**：
  1. 到达mainframe定义文件（def_*.sas）
  2. 到达源CSV/Excel文件
  3. 无法继续追踪（字段来源不明）
  4. 检测到循环依赖

### 5.2 文件处理工作流
```python
import asyncio
import re
from pathlib import Path
from langgraph.checkpoint.memory import InMemorySaver

def smart_chunk_sas_code(sas_code: str, max_tokens: int = 4000) -> list[str]:
    """
    智能分块SAS代码，按逻辑步骤（DATA步和PROC步）进行分割
    
    Args:
        sas_code: 完整的SAS代码字符串
        max_tokens: 每个块的最大token数限制
        
    Returns:
        分块后的SAS代码列表
    """
    # 移除注释以减少token使用（但保留原始代码用于最终分析）
    code_without_comments = re.sub(r'/\*.*?\*/', '', sas_code, flags=re.DOTALL)
    code_without_comments = re.sub(r'\*.*?;', '', code_without_comments)
    
    # 按DATA步和PROC步分割
    # 匹配 DATA ...; 和 PROC ...; 语句
    step_pattern = r'(data\s+\w+[^;]*;|proc\s+\w+[^;]*;)'
    steps = re.split(step_pattern, code_without_comments, flags=re.IGNORECASE)
    
    # 重新组合步骤标题和内容
    chunks = []
    current_chunk = ""
    
    for i in range(len(steps)):
        if i % 2 == 1:  # 步骤标题 (DATA或PROC)
            step_header = steps[i]
            step_content = steps[i + 1] if i + 1 < len(steps) else ""
            
            # 如果当前块加上新步骤会超过token限制，则开始新块
            potential_chunk = current_chunk + step_header + step_content
            if len(potential_chunk) > max_tokens * 4:  # 粗略估计token数（1 token ≈ 4字符）
                if current_chunk:
                    chunks.append(current_chunk)
                current_chunk = step_header + step_content
            else:
                current_chunk += step_header + step_content
        elif i == 0 and steps[i].strip():  # 开头的非步骤代码
            current_chunk += steps[i]
    
    if current_chunk:
        chunks.append(current_chunk)
    
    # 如果没有找到任何步骤，则按固定大小分块
    if not chunks:
        chunks = [sas_code[i:i + max_tokens * 4] for i in range(0, len(sas_code), max_tokens * 4)]
    
    return chunks

def trace_lineage_backwards(lineage_graph: dict, output_table: str, output_column: str) -> list[dict]:
    """
    从输出表字段反向追踪血缘关系
    
    Args:
        lineage_graph: 血缘关系图（由build_lineage_graph生成）
        output_table: 输出表名
        output_column: 输出字段名
        
    Returns:
        血缘路径列表，每个路径包含完整的映射信息
    """
    target_key = f"{output_table}.{output_column}"
    if target_key not in lineage_graph:
        # 如果找不到直接映射，返回空结果
        return [{
            "output_table": output_table,
            "output_column": output_column,
            "original_table": "",
            "original_column": "",
            "formula": "unknown",
            "script_file": ""
        }]
    
    results = []
    visited = set()  # 防止循环依赖
    
    def dfs(current_key: str, path: list):
        if current_key in visited:
            return  # 避免循环
        
        visited.add(current_key)
        
        if current_key not in lineage_graph:
            # 到达源字段（没有进一步的血缘关系）
            return
        
        for record in lineage_graph[current_key]:
            # 确保记录使用正确的字段名（output_table/output_column）
            updated_record = {
                "output_table": record.get("output_table", ""),
                "output_column": record.get("output_column", ""),
                "original_table": record.get("original_table", ""),
                "original_column": record.get("original_column", ""),
                "formula": record.get("formula", ""),
                "script_file": record.get("script_file", "")
            }
            results.append(updated_record)
            
            # 继续追踪源字段（如果源字段也是中间表）
            source_key = f"{record.get('original_table', '')}.{record.get('original_column', '')}"
            if source_key and source_key != current_key:
                dfs(source_key, path + [record])
    
    dfs(target_key, [])
    return results

import logging
logger = logging.getLogger(__name__)

async def analyze_sas_project(sas_code_dir: str, llm_config: dict):
    # 1. 创建LangGraph工作流
    lineage_workflow = create_sas_lineage_graph()
    checkpointer = InMemorySaver()
    
    # 2. 发现所有SAS文件（使用内置glob工具）
    sas_files = await glob("**/*.sas", path=sas_code_dir)
    logger.info(f"发现SAS文件数量: {len(sas_files)}")
    
    # 3. 批量分析SAS文件（智能分块处理）
    lineage_records = []
    batch_size = llm_config.get("max_files_per_batch", 5)
    
    for i in range(0, len(sas_files), batch_size):
        batch_files = sas_files[i:i + batch_size]
        batch_tasks = []
        
        for sas_file in batch_files:
            async def analyze_file(file_path):
                # 使用内置read_file工具读取文件
                content = await read_file(file_path)
                # 智能分块：按DATA/PROC步分割，而不是简单截断
                chunks = smart_chunk_sas_code(content, max_tokens=4000)
                file_records = []
                
                logger.info(f"开始分析SAS文件: {file_path}")
                logger.debug(f"分块数量: {len(chunks)}")
                for i, chunk in enumerate(chunks):
                    config = {"configurable": {"thread_id": f"{Path(file_path).stem}_chunk_{i}"}}
                    result = await lineage_workflow.ainvoke(
                        {
                            "file_path": file_path,
                            "sas_code": chunk,
                            "lineage_records": [],
                            "error": ""
                        },
                        config
                    )
                    file_records.extend(result.get("lineage_records", []))
                logger.info(f"提取血缘记录数: {len(file_records)}")
                
                return file_records
            
            batch_tasks.append(analyze_file(sas_file))
        
        # 并行处理批次
        batch_results = await asyncio.gather(*batch_tasks)
        for records in batch_results:
            lineage_records.extend(records)
    
    # 4. 构建完整的血缘图
    lineage_graph = build_lineage_graph(lineage_records)
    
    # 5. 识别输出CSV文件并进行反向追踪
    output_csvs = await glob("**/*.csv", path=sas_code_dir)
    final_lineage = []
    
    for output_csv in output_csvs:
        # 跳过血缘结果文件本身
        if "Output_sas_" in str(output_csv):
            continue
            
        table_name = Path(output_csv).stem
        try:
            columns = await get_csv_columns_async(output_csv)
            for column in columns:
                lineage_paths = trace_lineage_backwards(
                    lineage_graph, table_name, column
                )
                final_lineage.extend(lineage_paths)
        except pd.errors.EmptyDataError:
            print(f"警告: CSV文件为空 {output_csv}")
            continue
        except UnicodeDecodeError:
            print(f"警告: CSV文件编码问题 {output_csv}")
            continue
        except Exception as e:
            print(f"警告: 无法处理CSV文件 {output_csv}: {e}")
    
    return final_lineage

def build_lineage_graph(lineage_records):
    """构建血缘关系图"""
    graph = {}
    for record in lineage_records:
        # 统一使用output_table/output_column字段名
        target_key = f"{record['output_table']}.{record['output_column']}"
        if target_key not in graph:
            graph[target_key] = []
        graph[target_key].append(record)
    return graph

async def get_csv_columns_async(csv_file: str):
    """异步获取CSV列名"""
    import pandas as pd
    df = pd.read_csv(csv_file, nrows=0)
    return df.columns.tolist()
```

```python
SAS数据血缘分析主程序

import asyncio
import argparse
import yaml
from pathlib import Path
from src.cli import run_analysis

def main():
    parser = argparse.ArgumentParser(description="SAS数据血缘分析工具")
```
