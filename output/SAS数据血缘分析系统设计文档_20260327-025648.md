# SAS数据血缘分析系统技术设计文档

## 1. 项目概述

### 1.1 项目背景
SAS（Statistical Analysis System）是金融、医疗等行业的核心数据分析工具，但其代码复杂性导致数据血缘关系难以追踪。随着数据治理和合规要求的提升，需要自动化工具来分析SAS代码中的数据流转关系，建立从源表到目标表的完整血缘图谱。

### 1.2 项目目标
开发一个Python应用模块，能够：
- 分析SAS代码和相关数据集文件
- 生成数据血缘对照表，包含：源表名/文件名、源表字段、目标表名/文件名、目标表字段
- 支持从输出表反向追踪到源字段的完整路径
- 处理大规模SAS代码库，避免LLM上下文限制

### 1.3 核心需求
1. **输入支持**：mainframe SAS数据集、CSV文件、Excel文件、.sas7bdat中间文件
2. **输出格式**：CSV格式的数据血缘对照表
3. **分析入口**：从输出CSV文件开始反向分析
4. **处理规模**：支持大量SAS代码文件的分批处理
5. **技术栈**：Python 3.12、可配置LLM Provider、LangChain、Deep Agents SDK

## 2. 系统架构设计

### 2.1 整体架构
系统采用分层架构，包含以下核心组件：

```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│   文件发现层     │───▶│   预处理层       │───▶│   分析引擎层     │
└─────────────────┘    └──────────────────┘    └─────────────────┘
        │                       │                       │
        ▼                       ▼                       ▼
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│   存储管理层     │◀───│   结果聚合层     │◀───│   LLM推理层      │
└─────────────────┘    └──────────────────┘    └─────────────────┘
```

### 2.2 组件说明

#### 文件发现层（File Discovery Layer）
- 扫描指定目录，识别SAS代码文件（.sas）、输出CSV文件、中间.sas7bdat文件
- 建立文件依赖关系图，确定处理顺序
- 支持增量扫描，只处理新增或修改的文件

#### 预处理层（Preprocessing Layer）
- 解析SAS代码，提取关键语法结构（DATA、SET、MERGE、KEEP等语句）
- 识别输入/输出数据集映射关系
- 提取字段重命名、衍生字段表达式等元数据
- 将SAS代码转换为结构化中间表示

#### 分析引擎层（Analysis Engine Layer）
- 协调多Agent工作流
- 管理分析任务队列
- 处理跨文件血缘关系
- 实现反向追踪算法

#### LLM推理层（LLM Inference Layer）
- 使用Deep Agents SDK创建专用Agent
- 配置可插拔的LLM Provider
- 执行SAS代码语义分析
- 提取字段映射关系

#### 结果聚合层（Result Aggregation Layer）
- 合并多个分析结果
- 解决血缘关系冲突
- 生成标准化输出格式
- 支持增量更新

#### 存储管理层（Storage Management Layer）
- 缓存中间分析结果
- 持久化血缘关系数据
- 提供查询接口
- 支持多种存储后端（内存、SQLite、PostgreSQL）

### 2.3 数据流设计
1. **初始化阶段**：扫描目录，构建文件依赖图
2. **预处理阶段**：解析SAS代码，提取结构化元数据
3. **分析阶段**：按依赖顺序处理文件，使用LLM分析复杂逻辑
4. **聚合阶段**：合并结果，生成完整血缘图谱
5. **输出阶段**：导出CSV格式对照表

### 3.2 SAS解析模块（SASParserModule）

#### 功能职责
- 解析SAS代码语法
- 提取DATA/SET/MERGE/KEEP语句
- 识别字段重命名和衍生表达式
- 生成结构化中间表示
- **解析mainframe数据集定义文件**：从infile和input语句提取字段定义

#### 关键解析规则
- **DATA语句**：识别输出数据集名称
- **SET语句**：识别输入数据集，支持RENAME选项
- **MERGE语句**：识别多表连接，提取连接条件
- **KEEP语句**：识别输出字段列表
- **赋值语句**：识别衍生字段计算表达式
- **mainframe数据集定义**：解析infile和input语句中的字段定义

#### Mainframe数据集定义文件解析
根据参考文件`/input/sas/def_Churn.sas`和`/input/sas/def_train.sas`的实际格式：

1. **infile语句解析**：
   - 提取文件路径：`"&source_path/churn_raw&file_sfx"`
   - 提取记录格式：`recfm=F lrecl=657`

2. **input语句解析**：
   - 字段位置：使用`@`符号指定起始位置（如`@1`, `@3`）
   - 字段名称：可能包含空格，用单引号包围（如`'Loan Amount'`）
   - 字段类型和长度：使用格式说明符（如`PK2.0`, `S370FPD2.0`, `$EBCDIC3.`）

3. **解析示例**：
```sas
/* def_Churn.sas 示例 */
data &lib.. churn_raw /nolist;
    infile "&source_path/churn_raw&file_sfx" recfm=F lrecl=657;
    input
        @1    CustomerId                       PK2.0
        @3    Surname                           PK2.0
        @5    CreditScore                             PK3.0
        @8    Geography                              S370FPD2.0
        @10   Gender                       $EBCDIC3.
        /* ... 其他字段 ... */
    ;
run;
```

4. **解析结果**：
```json
{
  "dataset_name": "churn_raw",
  "source_file": "churn_raw",
  "record_length": 657,
  "fields": [
    {"name": "CustomerId", "position": 1, "format": "PK2.0"},
    {"name": "Surname", "position": 3, "format": "PK2.0"},
    {"name": "CreditScore", "position": 5, "format": "PK3.0"},
    {"name": "Geography", "position": 8, "format": "S370FPD2.0"},
    {"name": "Gender", "position": 10, "format": "$EBCDIC3."}
  ]
}
```

#### 中间表示格式
```json

  "input_datasets": ["churn_raw"],

      "output_field": "credit_scr",
```

### 3.3 多Agent分析模块（MultiAgentAnalysisModule）

#### 功能职责
- 创建两个专用Agent：SAS语法分析Agent和血缘关系提取Agent
- 协调Agent间的数据传递和任务分配
- 管理LLM上下文，避免超出限制
- 支持Agent间的协作和结果聚合

#### Agent设计
根据官方Deep Agents SDK文档（/langchain-ai/deepagents），正确的包名是`deepagents`，导入路径为`from deepagents import create_deep_agent`。创建两个专用Agent：

1. **SAS语法分析Agent**：专门解析SAS代码的语法结构（DATA、SET、MERGE、KEEP等语句）
2. **血缘关系提取Agent**：专门从语法分析结果中提取字段映射关系

#### Agent配置
```python
from deepagents import create_deep_agent
from typing import Dict, Any

class MultiAgentAnalysisModule:
    def __init__(self, llm_config: Dict[str, Any]):
        # 根据官方Deep Agents SDK文档，使用create_deep_agent函数创建Agent
        # 包名: deepagents (pip install deepagents)
        # 导入路径: from deepagents import create_deep_agent
        
        # SAS语法分析Agent：负责解析SAS代码的语法结构
        self.sas_parser_agent = create_deep_agent(
            model=llm_config.get("model", "openai:gpt-4o"),
            system_prompt="你是一个SAS代码专家，专门分析SAS语法结构，包括DATA、SET、MERGE、KEEP等语句",
            tools=[]  # 可根据需要添加自定义工具
        )
        
        # 血缘关系提取Agent：负责从解析结果中提取字段映射关系
        self.lineage_extractor_agent = create_deep_agent(
            model=llm_config.get("model", "openai:gpt-4o"),
            system_prompt="你是一个数据血缘专家，专门从SAS代码解析结果中提取源字段到目标字段的映射关系",
            tools=[]  # 可根据需要添加自定义工具
        )
        
    def analyze_sas_code(self, sas_code: str) -> Dict[str, Any]:
        """分析SAS代码并提取血缘关系
        
        Args:
            sas_code: 要分析的SAS代码字符串
            
        Returns:
            Dict包含解析结果和血缘关系映射
        """
        # 第一步：使用SAS解析Agent分析语法结构
        syntax_messages = [{"role": "user", "content": f"请分析以下SAS代码的语法结构:\n{sas_code}"}]
        syntax_result = self.sas_parser_agent.invoke({"messages": syntax_messages})
        
        # 第二步：使用血缘提取Agent分析字段映射
        lineage_messages = [{
            "role": "user", 
            "content": f"基于以下SAS语法分析结果，提取字段血缘关系:\n{syntax_result['messages'][-1].content}"
        }]
        lineage_result = self.lineage_extractor_agent.invoke({"messages": lineage_messages})
        
        return {
            "syntax_analysis": syntax_result["messages"][-1].content,
            "lineage_mapping": lineage_result["messages"][-1].content
        }
```

#### Agent协作流程
1. **任务分发**：主模块将SAS代码发送给SAS语法分析Agent
2. **语法解析**：SAS语法分析Agent识别DATA、SET、MERGE、KEEP等关键语句，返回结构化语法信息
3. **血缘提取**：血缘关系提取Agent接收语法分析结果，专门提取字段映射关系
4. **结果聚合**：主模块合并两个Agent的结果，生成完整的血缘关系数据

### 3.4 血缘追踪模块（LineageTrackingModule）

#### 功能职责
- 实现反向血缘追踪算法
- 处理跨文件血缘关系
- 解决循环依赖问题
- 生成完整血缘路径

#### 追踪算法
采用深度优先搜索（DFS）结合记忆化优化：

1. 从输出字段开始
2. 查找直接来源字段
3. 如果来源是中间表，递归追踪到源表
4. 缓存已解析的血缘关系，避免重复计算

### 3.5 结果管理模块（ResultManagementModule）

#### 功能职责
- 聚合分析结果
- 格式化输出数据
- 持久化血缘关系
- 提供查询接口

## 4. 数据血缘分析算法

### 4.1 字段映射识别算法

#### 直通字段识别
- 检查KEEP语句中的字段是否在SET语句的输入字段中存在
- 如果存在且无重命名，则为直通字段

#### 重命名字段识别
- 解析SET语句中的RENAME选项
- 建立源字段到目标字段的映射关系

#### 衍生字段识别
- 识别赋值语句（`field = expression`）
- 使用正则表达式提取表达式中的字段引用
- 分析表达式语义，确定操作类型

### 4.2 反向血缘追踪算法

#### 算法流程
```
function trace_lineage(output_table, output_field):
    if output_table is source table:
        return [(output_table, output_field)]
    
    # 查找生成该字段的SAS脚本
    script = find_script_for_table(output_table)
    field_mapping = get_field_mapping(script, output_field)
    
    if field_mapping.operation == "passthrough":
        return trace_lineage(field_mapping.input_table, field_mapping.input_field)
    elif field_mapping.operation == "rename":
        return trace_lineage(field_mapping.input_table, field_mapping.input_field)
    elif field_mapping.operation == "derived":
        sources = []
        for input_field in field_mapping.input_fields:
            sources.extend(trace_lineage(field_mapping.input_table, input_field))
        return sources
```

#### 循环依赖处理
- 维护访问标记，检测循环引用
- 设置最大递归深度限制
- 对循环依赖给出警告并截断追踪

### 4.3 多源合并处理

对于MERGE操作产生的字段：
- CustomerId等连接键：来自所有输入表
- 其他字段：来自特定输入表
- 衍生字段：可能引用多个输入表的字段

#### 文件处理实现方案
```python
import pandas as pd

# 读取SAS7BDAT文件
df = pd.read_sas("file.sas7bdat", format="sas7bdat")

# 读取CSV文件
df = pd.read_csv("file.csv", header=0, encoding="utf-8")

# 读取Excel文件（支持.xlsx和.xls格式）
df = pd.read_excel("file.xlsx", sheet_name=0, header=0)

# Excel与CSV/SAS文件处理差异：
# 1. Excel可能包含多个工作表，需要指定sheet_name参数
# 2. Excel文件通常有更复杂的格式（合并单元格、样式等），但pandas会自动处理基本数据
# 3. Excel文件编码问题较少，因为是二进制格式
# 4. 性能上Excel读取比CSV慢，但比SAS7BDAT快
```

- 缓存已分析的SAS代码结果
- 缓存血缘关系映射
- 支持增量更新

#### 并行处理
- 独立的SAS文件可以并行分析
- 使用线程池管理并发任务
- 限制并发数避免资源耗尽

- SAS语法解析错误：记录日志，跳过文件
