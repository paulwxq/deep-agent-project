# SAS数据血缘分析系统技术设计文档

## 1. 系统概述

### 1.1 项目背景
本系统旨在分析SAS代码和相关数据集，建立从源表字段到目标表字段的完整血缘关系映射。系统将处理多种数据源（mainframe SAS数据集、CSV文件、Excel文件）和输出格式（主要是CSV），并支持中间SAS数据集（.sas7bdat文件）的血缘追踪。

### 1.2 核心需求
- 以输出表作为分析入口，反向追踪每个结果字段的源字段
- 支持多级数据处理流程（一个SAS代码的输出作为另一个SAS代码的输入）
- 处理字段重命名、派生字段计算等转换逻辑
- 生成包含源表名/文件名、源字段、目标表名/文件名、目标字段的对照表

### 1.3 技术栈
- Python 3.12
- 可配置的LLM Provider
- LangChain-AI / Deep Agents SDK

## 2. 系统架构

### 2.1 整体架构
系统采用分层架构设计：
```
┌─────────────────┐
│   用户接口层     │
└────────┬────────┘
         │
┌────────▼────────┐
│   血缘分析引擎   │
└────────┬────────┘
         │
┌────────▼────────┐
│   数据解析层     │
└────────┬────────┘
         │
┌────────▼────────┐
│   存储与缓存层   │
└─────────────────┘
```

### 2.2 核心组件
1. **SAS代码解析器**：解析SAS代码，识别数据源、目标、字段映射和转换逻辑
2. **数据定义解析器**：解析mainframe数据定义文件（如def_Churn.sas），提取字段元数据
3. **血缘图构建器**：基于解析结果构建血缘关系图
4. **LLM辅助分析器**：处理复杂转换逻辑，使用LLM理解派生字段的计算规则
5. **结果生成器**：输出标准化的血缘对照表

## 3. 详细设计

### 3.1 SAS代码解析器

#### 3.1.1 功能描述
- 识别PROC IMPORT语句，提取源文件路径和目标数据集名
- 识别DATA语句，提取输入数据集和输出数据集
- 解析RENAME选项，建立字段重命名映射
- 识别KEEP/DROP语句，确定字段选择范围
- 提取派生字段的计算表达式

#### 3.1.2 解析规则
- **源数据识别**：
  - CSV/Excel源：通过PROC IMPORT的DATAFILE参数识别
  - Mainframe源：通过data语句中的infile路径识别
  - 中间SAS数据集：通过SET/MERGE语句识别

- **字段映射规则**：
  - 直接映射：字段未经过重命名或计算
  - 重命名映射：通过RENAME选项建立源字段到目标字段的映射
  - 派生字段：记录计算表达式和依赖的源字段

#### 3.1.3 字段映射类型定义

`mapping_type` 字段用于描述源字段到目标字段的转换类型，可选值及含义如下：

| 枚举值 | 含义 | 说明 |
|--------|------|------|
| `passthrough` | 直接传递 | 字段名称和值直接传递，无需重命名或计算 |
| `rename` | 重命名 | 字段含义不变，仅改变名称，需在formula中注明转换规则 |
| `derived` | 派生字段 | 通过计算、函数、条件逻辑生成的字段，formula记录实际表达式 |

#### 3.1.4 输出格式
```json
{
  "source_datasets": [
    {
      "name": "churn_raw",
      "type": "csv",
      "path": "/path/to/churn_modeling_1.csv",
      "fields": ["CustomerId", "CreditScore", "EstimatedSalary", "NumOfProducts"]
    }
  ],
  "target_dataset": {
    "name": "churn_out",
    "type": "csv",
    "path": "/path/to/churn_out.csv"
  },
  "field_mappings": [
    {
      "source_field": "CreditScore",
      "target_field": "credit_scr",
      "mapping_type": "rename",
      "formula": "rename CreditScore -> credit_scr"
    },
    {
      "source_field": "Balance/NumOfProducts",
      "target_field": "bal_per_prod",
      "mapping_type": "derived",
      "formula": "Balance / num_prod",
      "dependencies": ["Balance", "NumOfProducts"]
    },
    {
      "source_field": "Age",
      "target_field": "Age",
      "mapping_type": "passthrough",
      "formula": "passthrough"
    }
  ]
}
```

### 3.2 数据定义解析器

#### 3.2.1 功能描述
- 解析mainframe数据定义文件（如def_Churn.sas）
- 提取字段名称、位置、数据类型等元数据信息
- 处理特殊字符和空格在字段名中的情况

#### 3.2.2 解析规则
- 识别data语句中的数据集名称
- 解析input语句中的字段定义
- 处理带引号的字段名（如'Loan Amount'）
- 提取字段位置信息（@符号后的数字）

#### 3.2.3 输出格式
```json
{
  "dataset_name": "churn_raw",
  "source_type": "mainframe",
  "fields": [
    {
      "name": "CustomerId",
      "position": 1,
      "length": 2,
      "type": "numeric"
    },
    {
      "name": "CreditScore",
      "position": 5,
      "length": 3,
      "type": "numeric"
    }
  ]
}
```

### 3.3 血缘图构建器

#### 3.3.1 功能描述
- 基于SAS代码解析结果构建血缘关系图
- 处理多级数据流（链式依赖）
- 支持合并操作（MERGE）的血缘追踪
- 处理循环依赖检测

#### 3.3.2 图结构设计
- 节点：表示数据集（包含字段信息）
- 边：表示数据转换关系（包含字段映射信息）
- 属性：存储字段级别的映射详情

#### 3.3.3 算法流程
1. 初始化空血缘图
2. 遍历所有SAS代码文件，解析并添加节点和边
3. 对于每个输出数据集，执行反向遍历
4. 构建完整的字段级血缘路径

### 3.4 LLM辅助分析器

#### 3.4.1 功能描述
- 处理复杂的派生字段计算逻辑
- 解析难以用规则处理的转换表达式
- 提供自然语言解释的转换逻辑

#### 3.4.2 使用场景
- 复杂的数学表达式
- 条件逻辑（IF-THEN-ELSE）
- 函数调用和聚合操作
- 跨数据集的复杂合并逻辑

#### 3.4.3 Prompt设计
```text
你是一个SAS数据分析专家。请分析以下SAS代码片段中的派生字段计算逻辑：

{SAS_CODE_SNIPPET}

请以JSON格式返回分析结果，包含：
- 字段名称
- 计算表达式
- 依赖的源字段列表
- 转换逻辑的自然语言描述
```

### 3.5 结果生成器

#### 3.5.1 功能描述
- 生成标准化的血缘对照表
- 支持多种输出格式（CSV、JSON、数据库）
- 提供查询接口

#### 3.5.2 输出格式

血缘分析结果输出为CSV格式，包含以下6列，与需求示例`Output_sas_20260318.csv`保持一致：

| 列名 | 说明 | 示例 |
|------|------|------|
| `output_table` | 目标表名/文件名 | churn_out |
| `output_column` | 目标表字段名 | credit_scr |
| `original_table` | 源表名/文件名，多源合并时用分号分隔 | churn_raw 或 churn_raw; loan_raw |
| `original_column` | 源表字段名，多字段计算时用分号分隔 | CreditScore 或 Balance; NumOfProducts |
| `formula` | 转换逻辑说明 | rename CreditScore -> credit_scr 或 Balance / num_prod 或 passthrough |
| `script_file` | 生成该血缘关系的SAS脚本文件路径 | /path/to/01_churn_transform.sas |

**CSV示例**：

## 4. 数据流设计

### 4.1 处理流程
1. **文件发现**：扫描指定目录，识别SAS代码文件和数据定义文件
2. **并行解析**：同时解析多个SAS文件，提取元数据和转换逻辑
3. **血缘构建**：基于解析结果构建完整的血缘关系图
4. **结果输出**：生成用户友好的血缘对照表

### 4.2 LLM分块处理策略

由于实际场景中SAS代码文件较多，单次LLM调用无法承载完整上下文，系统采用以下分块处理策略：

#### 4.2.1 分块粒度设计

| 分块维度 | 粒度定义 | 阈值 |
|----------|----------|------|
| 按文件分块 | 单个SAS文件作为独立分析单元 | 文件行数 < 500行 |
| 按DATA步分块 | SAS文件内按DATA步/PROC步拆分 | 单一数据步代码 < 200行 |
| 按血缘阶段分块 | 按数据处理阶段拆分（如原始输入、中间处理、最终输出） | 阶段内依赖文件数 < 10个 |

**触发分块的场景**：
- 单个SAS文件超过500行
- 单个DATA步包含超过20个派生字段
- 存在跨文件的数据引用链（一个文件的输出作为另一个的输入）

#### 4.2.2 跨分块血缘合并算法

当分块处理后，需要将各块的解析结果合并为完整的血缘图：

```
输入：Block1_result, Block2_result, ..., BlockN_result
输出：MergedLineageGraph

算法步骤：
1. 去重合并：合并所有块的节点和边，过滤重复的字段映射
2. 依赖拓扑排序：对所有血缘关系进行拓扑排序，确保处理顺序正确
3. 中间表追踪：识别中间SAS数据集(.sas7bdat)，建立跨块引用链
4. 冲突解决：当同一目标字段存在多个来源时，优先保留最后执行的转换
5. 循环检测：检测并标记循环依赖关系
```

#### 4.2.3 LLM对话上下文管理

**滑动窗口策略**：
- 维护对话上下文历史窗口，窗口大小设置为最近5次有效的LLM调用记录
- 每完成一次成功解析，将结果摘要追加到上下文，新请求前先压缩历史摘要

**上下文压缩规则**：
- 已成功解析的简单字段映射（passthrough/rename）压缩为一行摘要
- 仅保留派生字段的完整计算表达式
- 中间结果使用占位符引用（如 `Block3_Result: field_X`）

**示例对话序列**：
```
[请求1] 分析 01_churn_transform.sas → [结果1] 提取15个字段映射
[请求2] 分析 02_loan_transform.sas → [结果2] 提取12个字段映射
[请求3] 分析 03_merge_transform.sas（引用了前两个输出）
     → 上下文包含：结果1和结果2的摘要 + 03_merge_transform.sas的完整代码
     → LLM能够理解跨文件的字段引用关系
```

#### 4.2.4 示例场景

**场景1：大型SAS文件处理**
```
输入：train_model.sas（1200行代码，包含3个DATA步）
分块策略：
  - Block A: DATA step_1 (350行) → 输出 temp_step1.sas7bdat
  - Block B: DATA step_2 (400行) → 输入为temp_step1，输出 temp_step2.sas7bdat
  - Block C: DATA step_3 (450行) → 输入为temp_step2，输出 final_output.csv

合并结果：三个Block的血缘链合并，形成从raw_input到final_output的完整血缘图
```

**场景2：多文件依赖链**
```
输入：20个相互引用的SAS文件，文件总行数约8000行
分块策略：
  - 按数据流拓扑排序，先处理无依赖的文件（5个）
  - 再处理依赖层级为1的文件（8个）
  - 最后处理依赖层级为2的文件（7个）

LLM调用策略：
  - 每批次最多处理3个文件
  - 每批次建立独立对话上下文
  - 批次间通过持久化缓存传递中间结果
```

**场景3：复杂派生字段识别**
```
单个字段涉及复杂的CASE WHEN逻辑
输入：IF Age >= 60 THEN senior_flag = 1; ELSE IF Age >= 40 THEN senior_flag = 0; ...
分块处理：
  - 规则引擎：识别简单赋值和RENAME
  - LLM处理：识别复杂条件逻辑中的依赖字段
  - 合并输出：senior_flag → derived → (Age >= 60) 依赖字段: Age
```

#### 5.2.1 异常定义

API层应定义以下自定义异常类型：

| 异常类型 | 说明 | 触发场景 |
|----------|------|----------|
| `LineageParseError` | SAS代码解析失败 | 语法错误、无法识别的语句结构 |
| `LineageFileNotFoundError` | 引用的数据文件不存在 | CSV/Excel文件路径无效 |
| `LineageFieldNotFoundError` | 字段引用不存在 | 源字段在定义中找不到 |
| `LineageCircularDependencyError` | 存在循环依赖 | 数据流中存在循环引用 |
| `LineageLLMError` | LLM调用失败 | API超时、限流或返回格式错误 |

```python
class LineageError(Exception):
    """血缘分析基础异常类"""
    def __init__(self, message: str, details: dict = None):
        self.message = message
        self.details = details or {}
        super().__init__(self.message)

class LineageParseError(LineageError):
    """SAS代码解析异常"""
    pass

class LineageFileNotFoundError(LineageError):
    """文件未找到异常"""
    pass

class LineageFieldNotFoundError(LineageError):
    """字段未找到异常"""
    pass

    """循环依赖异常"""
    pass

class LineageLLMError(LineageError):
    """LLM调用异常"""
```

## 6. 错误处理与容错

### 6.1 错误类型
- **语法错误**：SAS代码语法不正确
- **文件缺失**：引用的数据文件不存在
- **字段不匹配**：源字段在目标中找不到对应
- **循环依赖**：数据流中存在循环引用

### 6.2 处理策略
- **优雅降级**：部分文件解析失败不影响整体分析
- **错误日志**：详细记录解析错误和警告信息
- **人工干预点**：标记需要人工确认的复杂转换逻辑
- **重试机制**：对LLM调用失败的情况进行重试
