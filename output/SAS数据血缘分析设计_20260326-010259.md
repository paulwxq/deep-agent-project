# SAS数据血缘分析系统技术设计文档

## 1. 概述

### 1.1 项目背景
本项目旨在构建一个Python应用模块，用于分析SAS代码和相关数据集，生成数据血缘关系对照表。通过分析SAS脚本中的数据处理逻辑，建立源字段与目标字段之间的映射关系，为数据治理、影响分析和合规审计提供支持。

### 1.2 核心需求
- 以输出表作为分析入口，反向追踪每个结果字段的源字段
- 支持多种数据源类型：mainframe SAS数据集、CSV文件、Excel文件
- 处理中间临时SAS文件（.sas7bdat格式）
- 生成包含源表名/文件名、源表字段、目标表名/文件名、目标表字段的血缘对照表
- 支持大规模SAS代码库的分析，避免单次处理过多内容导致上下文溢出

## 2. 系统架构

### 2.1 整体架构
系统采用分层架构设计，包含以下核心组件：

```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│   文件发现层     │───▶│   代码解析层      │───▶│   血缘构建层     │
└─────────────────┘    └──────────────────┘    └─────────────────┘
       │                        │                        │
       ▼                        ▼                        ▼
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│   存储管理层     │◀───│   LLM分析层       │───▶│   输出管理层     │
└─────────────────┘    └──────────────────┘    └─────────────────┘
```

### 2.2 技术栈
- **编程语言**: Python 3.12
- **LLM框架**: LangChain + Deep Agents SDK
- **可配置LLM Provider**: 支持多种大语言模型提供商
- **文件处理**: pandas（CSV/Excel）、pyreadstat（SAS文件）
- **并发处理**: asyncio + ThreadPoolExecutor

#### 3.1.3 CSV输入/输出区分策略
由于SAS代码的输出文件都是CSV格式，需要通过分析脚本内容来区分CSV文件的输入/输出角色：

**区分规则**：
1. **通过PROC EXPORT定位输出CSV**：解析所有transform脚本，识别`PROC EXPORT ... OUTFILE="path"`语句，路径指向的CSV标记为输出文件
2. **通过INFILE定位输入CSV**：解析所有transform脚本，识别`INFILE "path"`语句，路径指向的CSV标记为输入文件
3. **排除法确认**：扫描完成后，未被标记为输入或输出的CSV文件，检查是否被transform脚本引用（通过文件名匹配）
4. **中间CSV处理**：如果CSV同时被多个脚本引用（某些作为输入，某些作为输出），以最终输出端的角色为准

**实现逻辑**：
```python
def classify_csv_files(self, scripts: List[str]) -> Dict[str, str]:
    """
    分析脚本内容，分类CSV文件类型
    返回: {csv文件路径: 'input'|'output'|'intermediate'}
    """
    csv_files = {}
    for script in scripts:
        script_content = self._read_script(script)
        # 提取PROC EXPORT中的OUTFILE路径
        export_paths = self._extract_export_paths(script_content)
        for path in export_paths:
            csv_files[path] = 'output'
        # 提取INFILE中的输入路径
        infile_paths = self._extract_infile_paths(script_content)
        for path in infile_paths:
            if path not in csv_files:
                csv_files[path] = 'input'
    return csv_files

def _extract_export_paths(self, content: str) -> List[str]:
    """使用正则提取PROC EXPORT的OUTFILE路径"""
    pattern = r'OUTFILE\s*=\s*["\']([^"\']+)["\']'
    return re.findall(pattern, content, re.IGNORECASE)

def _extract_infile_paths(self, content: str) -> List[str]:
    """使用正则提取INFILE路径"""
    pattern = r'INFILE\s+["\']([^"\']+)["\']'
    return re.findall(pattern, content, re.IGNORECASE)
```

#### 3.2.4 脚本依赖分析
由于存在"中间结果"场景（一个脚本的输入是另一个脚本的输出），需要建立脚本间的依赖关系图：

**依赖关系识别规则**：
1. **输出CSV → 脚本映射**：记录每个输出CSV由哪个脚本生成
2. **脚本输入 → 前置脚本映射**：如果脚本A的输入CSV恰好是脚本B的输出CSV，则A依赖B
3. **数据集依赖**：如果脚本引用.sas7bdat临时文件，需要追踪该文件的生成脚本

**依赖图构建**：
```python
class ScriptDependencyGraph:
    def __init__(self):
        # 脚本 → 输出CSV列表
        self.script_outputs: Dict[str, List[str]] = {}
        # 脚本 → 输入源列表（CSV或数据集）
        self.script_inputs: Dict[str, List[str]] = {}
        # 脚本 → 直接依赖的脚本列表
        self.dependencies: Dict[str, List[str]] = {}
        # 执行拓扑排序后的脚本列表（从源头到末端）
        self.execution_order: List[str] = []
        # CSV → 生成脚本的映射
        self.csv_to_script: Dict[str, str] = {}
    
    def build_graph(self, scripts: List[str], csv_classification: Dict[str, str]):
        """
        构建脚本依赖图
        csv_classification: {csv路径: 'input'|'output'|'intermediate'}
        """
        for script in scripts:
            inputs = self._extract_input_sources(script)
            self.script_inputs[script] = inputs
            
            # 查找输入CSV是由哪个脚本生成的
            script_deps = []
            for inp in inputs:
                if inp in self.csv_to_script:
                    script_deps.append(self.csv_to_script[inp])
            self.dependencies[script] = list(set(script_deps))
            
            # 记录脚本输出和映射
            outputs = self._extract_output_csvs(script)
            self.script_outputs[script] = outputs
            for out_csv in outputs:
                self.csv_to_script[out_csv] = script
        
        # 执行拓扑排序（Kahn算法）
        self.execution_order = self._topological_sort()
        return self
    
    def _extract_input_sources(self, script: str) -> List[str]:
        """提取脚本的输入源（CSV文件或数据集）"""
        content = self._read_script(script)
        sources = []
        # INFILE路径
        infile_pattern = r'INFILE\s+["\']([^"\']+)["\']'
        sources.extend(re.findall(infile_pattern, content, re.IGNORECASE))
        # SET语句中的数据集
        set_pattern = r'\bSET\s+([&.\w]+)'
        sources.extend(re.findall(set_pattern, content, re.IGNORECASE))
        return sources
    
    def _extract_output_csvs(self, script: str) -> List[str]:
        """提取脚本的输出CSV文件"""
        content = self._read_script(script)
        export_pattern = r'OUTFILE\s*=\s*["\']([^"\']+)["\']'
        return re.findall(export_pattern, content, re.IGNORECASE)
    
    def _topological_sort(self) -> List[str]:
        """Kahn算法实现拓扑排序"""
        # 计算每个脚本的入度（被依赖数）
        in_degree = {s: len([d for deps in self.dependencies.values() if s in deps]) 
                     for s in self.script_outputs}
        
        # 从入度为0的脚本开始（源头脚本）
        queue = [s for s, deg in in_degree.items() if deg == 0]
        result = []
        
        while queue:
            node = queue.pop(0)
            result.append(node)
            # 减少依赖该节点的脚本的入度
            for other, deps in self.dependencies.items():
                if node in deps:
                    in_degree[other] -= 1
                    if in_degree[other] == 0:
                        queue.append(other)
        
        # 检测循环依赖
        if len(result) != len(self.script_outputs):
            unresolved = set(self.script_outputs.keys()) - set(result)
            raise CircularDependencyError(
                f"检测到循环依赖，涉及脚本: {unresolved}"
            )
        
        return result
```

**依赖分析执行流程**：
1. 首先调用`FileDiscoveryManager.discover_files()`获取所有文件
2. 调用`FileDiscoveryManager.classify_csv_files()`分类CSV类型
3. 构建`csv_to_script`映射：遍历所有脚本，将PROC EXPORT的OUTFILE CSV映射到脚本
4. 调用`ScriptDependencyGraph.build_graph()`构建依赖图
5. 血缘分析时，按`execution_order`顺序处理，确保依赖脚本先分析完成

### 3.5 血缘关系构建模块

#### 3.5.1 功能描述
整合各模块的分析结果，构建完整的血缘关系图，并生成最终的对照表。

#### 3.5.2 血缘构建算法
采用递归反向追踪算法，从输出字段出发，逐层向上寻找源字段。

**算法伪代码**：
```
function build_lineage(target_field, target_script):
    lineage = []
    visited = set()  # 防止循环追溯
    
    function trace(field, script, depth):
        # 递归终止条件1：超过最大深度
        if depth > MAX_TRACE_DEPTH:
            return [(field, script, "MAX_DEPTH_EXCEEDED")]
        
        # 递归终止条件2：已访问过的节点
        node_key = (field, script)
        if node_key in visited:
            return [(field, script, "CYCLE_DETECTED")]
        
        visited.add(node_key)
        
        # 步骤1：解析当前脚本，查找字段来源
        script_info = parse_script(script)
        
        # 步骤2：检查是否是直接输入字段（PASSTHROUGH）
        if field in script_info.input_fields:
            return [(field, script_info.input_source, "PASSTHROUGH")]
        
        # 步骤3：检查是否是重命名字段（RENAME）
        if field in script_info.rename_map:
            source_field = script_info.rename_map[field]
            return trace(source_field, script_info.input_script, depth + 1)
        
        # 步骤4：检查是否是衍生字段（DERIVED）
        if field in script_info.derived_fields:
            # 衍生字段可能来自多个源
            sources = script_info.derived_fields[field].source_fields
            return [(src, script_info.input_script, "DERIVED") 
                    for src in sources]
        
        # 递归终止条件3：无法追溯的字段
        return [(field, script, "UNTRACED")]
    
    return trace(target_field, target_script, 0)
```

**递归终止条件**：
1. **MAX_DEPTH_EXCEEDED**：追踪深度超过阈值（默认10层），防止无限递归
2. **CYCLE_DETECTED**：检测到循环依赖（字段A依赖B，B依赖A），记录异常并返回
3. **UNTRACED**：字段无法追溯到任何源（如硬编码常量、系统函数）
4. **PASSTHROUGH**：字段直接来自输入数据集，未经过任何转换

#### 3.5.3 字段匹配规则
字段名匹配采用三级策略：

**第一级：精确匹配**
```python
def exact_match(field: str, candidates: List[str]) -> Optional[str]:
    """精确匹配，不区分大小写"""
    field_lower = field.lower()
    for cand in candidates:
        if cand.lower() == field_lower:
            return cand
    return None
```

**第二级：模糊匹配（Levenshtein距离）**
```python
def fuzzy_match(field: str, candidates: List[str], threshold: float = 0.8) -> Optional[str]:
    """基于编辑距离的模糊匹配"""
    from difflib import SequenceMatcher
    field_lower = field.lower()
    best_match = None
    best_ratio = 0.0
    for cand in candidates:
        ratio = SequenceMatcher(None, field_lower, cand.lower()).ratio()
        if ratio > best_ratio and ratio >= threshold:
            best_ratio = ratio
            best_match = cand
    return best_match
```

**第三级：拼写纠错映射**
```python
def spell_correct(field: str, correction_map: Dict[str, str]) -> str:
    """应用拼写纠错映射"""
    return correction_map.get(field, field)
```

**匹配流程**：精确匹配成功 → 返回结果；精确匹配失败 → 尝试模糊匹配 → 尝试拼写纠错 → 记录为"UNMATCHED"。

```python
        self.max_depth = config.max_trace_depth
        self.fuzzy_threshold = config.fuzzy_match_threshold
        self.spell_corrections = config.spell_corrections
        self.lineage_records: List[FieldLineage] = []
        self.visited: Set[str] = set()  # (table, field, script) -> status
    
    def build_lineage_from_output(self, output_dataset: str) -> List[FieldLineage]:
        """从输出数据集开始反向构建血缘"""
        # 1. 获取输出数据集的字段列表
        output_fields = self._get_dataset_fields(output_dataset)
        
        # 2. 获取生成该输出的脚本
        generating_script = self._find_generating_script(output_dataset)
        
        # 3. 对每个字段进行追溯
        for field in output_fields:
            self._trace_field(field, output_dataset, generating_script, depth=0)
        
        return self.lineage_records
```

### 4.4 配置文件加载器
```python
import yaml
import os
from typing import Dict, Any
from pathlib import Path

class ConfigLoader:
    """配置文件加载器，支持环境变量替换"""
    
    @staticmethod
    def load_config(config_path: str) -> Dict[str, Any]:
        """加载并解析配置文件"""
        with open(config_path, 'r', encoding='utf-8') as f:
            content = f.read()
        
        # 替换环境变量
        content = ConfigLoader._substitute_env_vars(content)
        
        # 解析YAML
        config = yaml.safe_load(content)
        return config
    
    @staticmethod
    def _substitute_env_vars(content: str) -> str:
        """替换${ENV_VAR}格式的环境变量"""
        import re
        pattern = r'\$\{([^}]+)\}'
        
        def replace(match):
            env_var = match.group(1)
            return os.environ.get(env_var, match.group(0))
        
        return re.sub(pattern, replace, content)
    
    @staticmethod
    def merge_with_defaults(config: Dict) -> Dict:
        """合并用户配置与默认配置"""
        defaults = {
            'llm': {
                'provider': 'deepseek',
                'model': 'deepseek-chat',
                'temperature': 0.0,
                'max_tokens': 4096,
                'timeout': 120
            },
            'lineage': {
                'max_trace_depth': 10,
                'fuzzy_match_threshold': 0.8,
                'confidence_threshold': 0.7
            },
            'spell_corrections': {},
            'paths': {
                'root_dir': './input/sas',
                'output_dir': './output',
                'cache_dir': './.cache'
            },
            'logging': {
                'level': 'INFO',
                'format': '%(asctime)s - %(name)s - %(levelname)s - %(message)s',
                'file': './logs/lineage_analysis.log'
            }
        }
        
        # 深度合并
        def deep_merge(base: Dict, override: Dict) -> Dict:
            result = base.copy()
            for key, value in override.items():
                if key in result and isinstance(result[key], dict) and isinstance(value, dict):
                    result[key] = deep_merge(result[key], value)
                else:
                    result[key] = value
            return result
        
        return deep_merge(defaults, config)
```

## 5. 数据处理流程

### 4.1 主流程
1. **文件发现**: 扫描目录，分类所有相关文件
2. **预处理**: 解析数据集定义文件，建立字段标准化映射
3. **脚本分析**: 逐个分析SAS转换脚本，提取基础血缘信息
4. **LLM增强**: 对复杂逻辑使用LLM进行深度分析
5. **血缘构建**: 整合所有信息，构建完整的血缘关系
6. **结果输出**: 生成标准化的血缘对照表

### 4.2 异常处理
- **字段名不匹配**: 使用模糊匹配和拼写纠错
- **文件缺失**: 记录警告，继续处理其他文件
- **LLM分析失败**: 回退到规则引擎，标记低置信度
- **循环依赖**: 检测并中断循环，记录异常

## 5. 输出格式

### 5.1 标准输出格式
最终输出为CSV格式，包含以下核心字段：

| 字段名 | 说明 | 示例 |
|--------|------|------|
| source_table | 源表名或文件名 | churn_raw, train_1.csv |
| source_field | 源表字段名 | CreditScore, EstimatedSalary |
| target_table | 目标表名或文件名 | churn_out, merged_out |
| target_field | 目标表字段名 | credit_scr, est_sal |

### 5.2 扩展信息（可选）
系统内部可维护以下扩展信息，用于调试和增强分析：
- **transformation_type**: 转换类型（passthrough, rename, derived, merge）
- **script_file**: 源SAS脚本文件路径
- **confidence_score**: 血缘关系置信度
