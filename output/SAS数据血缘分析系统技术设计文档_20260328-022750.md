# SAS数据血缘分析系统技术设计文档

## 1. 项目概述

### 1.1 项目背景
本项目旨在开发一个Python应用模块，用于分析SAS代码和相关数据集，生成数据血缘分析对照表。该系统将帮助数据治理团队理解数据流转过程，追踪字段级别的数据来源，满足数据合规性和可追溯性要求。

### 1.2 核心需求
- **输入源**：SAS代码文件（.sas）、源数据（mainframe SAS数据集、CSV、Excel文件）
- **输出目标**：CSV格式的输出文件
- **中间产物**：SAS临时文件（.sas7bdat后缀）
- **分析方向**：从输出表作为入口，反向追踪每个目标字段对应的源字段
- **输出格式**：源表名/文件名、源表字段、目标表名/文件名、目标表字段

### 1.3 技术约束
- Python 3.12 运行环境
- 支持可配置的LLM Provider（Anthropic、OpenAI、Google等）
- 利用LangChain或Deep Agents SDK构建智能分析能力
- 处理大规模SAS代码集合（不能一次性全部提交给LLM）
- 支持增量处理和缓存机制以优化性能

## 2. 系统架构设计

系统采用分层架构设计，包含以下核心组件：

### 2.2 设计原则
- **输出驱动**：从输出CSV文件作为入口，反向追踪血缘关系
- **增量处理**：支持分批处理大量SAS文件，避免LLM上下文限制
- **上下文隔离**：每个分析任务保持独立上下文，防止信息污染
- **缓存优化**：利用文件系统缓存中间结果，避免重复分析
- **可扩展性**：模块化设计，便于添加新的解析规则和数据源类型

## 3. 核心模块设计

### 3.1 SAS代码解析器 (SASCodeParser)
负责解析SAS代码文件，提取关键信息：

**功能特性：**
- 识别`PROC IMPORT`语句，提取源文件路径和数据集名称
- 解析`DATA`语句，识别输入数据集和输出数据集
- 提取`RENAME`操作，建立字段映射关系
- 识别`KEEP`/`DROP`语句，确定字段选择范围
- 解析`MERGE`/`JOIN`操作，处理多源数据合并场景
- 提取`PROC EXPORT`语句，确定最终输出文件
- 处理SAS宏变量（`%LET`、`&var`），解析动态数据集名称
- 识别注释掉的代码块，避免误解析

**输出结构：**
```python
class SASCodeInfo:
    source_datasets: List[DatasetRef]  # 源数据集引用
    target_dataset: DatasetRef         # 目标数据集
    field_mappings: Dict[str, FieldMapping]  # 字段映射关系
    operations: List[Operation]        # 数据操作序列
    libname_mappings: Dict[str, str]   # libname到物理路径的映射
```

### 3.2 Mainframe数据集定义解析器 (MainframeDefParser)
专门解析Mainframe数据集定义文件（如`def_Churn.sas`、`def_train.sas`），这些文件使用`infile`和`input`语句定义字段结构。

**功能特性：**
- 解析`infile`语句，提取源文件路径（支持宏变量如`&source_path`）
- 解析`input`语句中的字段定义，包括字段位置（`@1`）、字段名、数据类型和格式
- 处理带空格的字段名（如`'Loan Amount'`），正确提取字段标识符
- 识别`data &lib..dataset_name`语法，提取逻辑数据集名称
- 建立Mainframe源文件到逻辑数据集的映射关系

**输出结构：**
```python
class MainframeDefInfo:
    dataset_name: str                  # 逻辑数据集名称（如churn_raw）
    source_file_path: str             # 物理源文件路径
    fields: List[MainframeFieldDef]   # 字段定义列表
    
class MainframeFieldDef:
    position: int                     # 字段在记录中的起始位置
    name: str                         # 字段名称
    format: str                       # SAS格式说明（如PK2.0、S370FPD6.0）
    is_quoted: bool                   # 字段名是否用引号包围
```

### 3.3 输出文件发现器 (OutputFileDiscoverer)
负责扫描和索引所有输出CSV文件，建立输出文件到SAS脚本的映射关系。

**功能特性：**
- 递归扫描指定目录下的所有CSV文件
- 解析CSV文件内容，识别血缘分析结果格式
- 建立输出表名到生成脚本的映射关系
- 支持增量发现，只处理新增或修改的输出文件

**输出结构：**
```python
class OutputFileInfo:
    output_table: str                 # 输出表名
    script_file: str                  # 生成该输出的SAS脚本路径
    file_path: str                    # CSV文件完整路径
```

### 3.2 数据血缘追踪器 (DataLineageTracker)
基于解析结果构建完整的血缘关系图：

**核心算法：**
- **正向构建**：从源数据开始，逐层构建数据流图
- **反向查询**：从目标字段出发，递归查找所有源字段
- **跨文件关联**：通过中间SAS文件(.sas7bdat)连接不同SAS脚本

**血缘图结构：**
```python
class LineageNode:
    dataset_name: str
    field_name: str
    node_type: Literal["source", "intermediate", "target"]
    
class LineageEdge:
    source: LineageNode
    target: LineageNode
    transformation: str  # 转换逻辑描述
```

### 3.3 LLM增强分析器 (LLMEnhancedAnalyzer)
处理复杂转换逻辑和模糊场景：

**应用场景：**
- 解析复杂的表达式和函数调用
- 处理动态字段生成（如宏变量展开）
- 推断隐式字段依赖关系
- 验证和修正自动解析结果

**实现策略：**
- 使用Deep Agents框架构建专用分析代理
- 采用FilesystemBackend管理分析上下文
- 实现子代理分工：表达式解析代理、依赖推断代理、验证代理

### 3.4 任务编排器 (TaskOrchestrator)
管理大规模分析任务的执行流程：

**工作流程：**
1. 扫描输入目录，识别所有SAS文件和数据文件
2. 构建文件依赖图，确定分析顺序
3. 分批处理文件组，每批控制在LLM上下文限制内
4. 聚合各批次结果，生成完整血缘图
5. 输出标准化的血缘对照表

## 4. 数据血缘分析算法

### 4.1 输出文件发现与脚本定位
系统采用"输出驱动"的工作流程，首先发现所有输出CSV文件并定位对应的SAS脚本：

1. **输出文件扫描**：递归扫描指定目录，发现所有CSV格式的输出文件
2. **脚本映射建立**：通过文件命名约定或配置文件建立输出表到SAS脚本的映射
3. **增量处理**：记录已处理的输出文件，支持增量分析
4. **入口准备**：为每个输出表准备反向追踪的入口点

### 4.2 Mainframe数据集定义解析
专门处理`def_xxx.sas`类型的Mainframe数据集定义文件：

1. **infile语句解析**：提取源文件路径（支持宏变量如`&source_path`）
2. **input语句解析**：解析字段位置、名称、格式信息
3. **字段名处理**：正确处理带空格的字段名（如`'Loan Amount'`）
4. **数据集名提取**：从`data &lib..dataset_name`语法中提取逻辑数据集名称

### 4.3 基础SAS语法解析
**步骤1：文件类型识别**
- CSV/Excel文件 → 源数据
- Mainframe定义文件（def_xxx.sas）→ Mainframe源数据定义
- .sas7bdat文件 → 中间数据
- .sas文件 → 转换逻辑
- 输出CSV → 目标数据

**步骤2：SAS语法解析**
```python
def parse_sas_file(file_path: str) -> SASCodeInfo:
    # 1. 提取PROC IMPORT语句和注释掉的代码
    imports = extract_import_statements(content)
    
    # 2. 提取DATA语句块，处理宏变量
    data_blocks = extract_data_blocks(content)
    
    # 3. 解析每个DATA块
    for block in data_blocks:
        input_datasets = parse_input_datasets(block)
        output_dataset = parse_output_dataset(block)
        field_mappings = parse_field_operations(block)
        libname_mappings = parse_libname_statements(block)
        
    # 4. 提取PROC EXPORT语句
    exports = extract_export_statements(content)
    
    return SASCodeInfo(...)
```

**步骤3：字段映射构建**
- 直接赋值：`new_field = old_field`
- 重命名：`RENAME=(old_name = new_name)`
- 表达式：`derived_field = expr(old_field1, old_field2, ...)`
- 聚合操作：`SUM(field)`, `MEAN(field)`等

### 4.2 反向血缘追踪算法
**算法流程：**
```
function trace_lineage(target_field, target_dataset):
    if target_dataset is source:
        return [(target_dataset, target_field)]
    
    # 查找生成该字段的SAS代码
    sas_code = find_sas_code_for_dataset(target_dataset)
    field_mapping = sas_code.field_mappings[target_field]
    
    if field_mapping.is_direct_copy():
        source_field = field_mapping.source_field
        source_dataset = field_mapping.source_dataset
        return trace_lineage(source_field, source_dataset)
    
    elif field_mapping.is_expression():
        dependencies = extract_field_dependencies(field_mapping.expression)
        result = []
        for dep_field, dep_dataset in dependencies:
            result.extend(trace_lineage(dep_field, dep_dataset))
        return result
    
    else:
        # 使用LLM分析复杂场景
        return llm_analyze_complex_dependency(field_mapping)
```

### 4.4 多文件关联算法
**libname和宏变量解析：**
1. 解析`libname`语句，建立逻辑库名到物理路径的映射
2. 处理宏变量（如`&lib`、`&source_path`），动态解析数据集引用
3. 支持`data &lib..dataset_name`格式的数据集名称解析

**依赖图构建：**
1. 为每个SAS文件创建节点
2. 基于输入/输出数据集建立边连接
3. 识别循环依赖并标记警告
4. 确定拓扑排序的分析顺序

**中间文件处理：**
- .sas7bdat文件作为连接点
- 通过逻辑数据集名称（而非文件名）建立跨脚本连接
- 处理临时文件的生命周期管理

**血缘路径剪枝：**
- 当追踪到已知的源表（Mainframe定义或CSV/Excel）时，停止递归
- 避免重复分析相同的血缘路径
- 显著提升大规模血缘分析的效率

## 5. 技术实现细节

### 5.1 Deep Agents集成方案
使用Deep Agents SDK构建智能分析系统。基于Context7文档中的最佳实践，采用FilesystemBackend管理分析上下文，并通过子代理分工处理复杂场景：

```python
from deepagents import create_deep_agent
from deepagents.backends import FilesystemBackend

def create_lineage_agent():
    # 创建主分析代理
    agent = create_deep_agent(
        model="anthropic:claude-sonnet-4-5-20250929",
        memory=["./prompts/system_prompt.md"],
        skills=["./skills/sas_parsing/", "./skills/lineage_analysis/"],
        backend=FilesystemBackend(root_dir="./workspace"),
        subagents=[
            {
                "name": "expression_parser",
                "description": "解析SAS表达式中的字段依赖关系",
                "system_prompt": "你专门分析SAS表达式，识别所有输入字段和函数依赖...",
                "tools": [parse_expression_tool]
            },
            {
                "name": "dependency_validator", 
                "description": "验证血缘关系的正确性和完整性",
                "system_prompt": "你验证字段血缘关系是否合理，检查是否存在循环依赖或缺失源字段...",
                "tools": [validate_lineage_tool]
            }
        ]
    )
    return agent
```

### 5.2 LangChain替代方案
如果选择LangChain框架，可利用FilesystemFileSearchMiddleware提供文件搜索能力，适用于需要在大量SAS文件中定位特定代码片段的场景：

```python
from langchain.agents import create_agent
from langchain.agents.middleware import FilesystemFileSearchMiddleware

agent = create_agent(
    model="gpt-4o-mini",
    tools=[sas_parse_tool, lineage_trace_tool],

            root_path="./sas_project",
            use_ripgrep=True,
            max_file_size_mb=50
        )
    ]
)
```

### 5.3 关键数据结构
**血缘关系记录：**
```python
@dataclass
class LineageRecord:
    output_table: str      # 输出表名（对应需求中的output_table）
    output_column: str     # 输出字段名（对应需求中的output_column）
    original_table: str    # 源表名/文件名（对应需求中的original_table）
    original_column: str   # 源字段名（对应需求中的original_column）
    formula: str           # 转换逻辑描述（对应需求中的formula）
    script_file: str       # 生成该输出的SAS脚本路径（对应需求中的script_file）
    confidence: float = 1.0  # 置信度 (0.0-1.0)，默认为1.0表示确定性解析
```

**配置文件格式：**
```yaml

  provider: "anthropic"

```

### 5.4 错误处理与容错机制
- **语法错误**：跳过无法解析的SAS代码块，记录警告日志
- **文件缺失**：当SAS脚本中引用的数据集定义文件不存在时，标记为"unknown source"
- **字段追踪失败**：当字段血缘无法追踪到最终源表时，仍然输出该记录，但标记为"unknown source"
- **LLM失败**：降级到规则引擎，使用保守的血缘推断
- **内存溢出**：自动清理缓存，分批处理大型项目
- **部分结果输出**：即使某些字段无法完全追踪，也输出已知的血缘关系，帮助用户识别分析盲区
