# SAS数据血缘分析工具 - 技术设计文档

## 1. 项目概述

### 1.1 项目背景
本项目旨在开发一个独立的Python工具，用于分析SAS代码中的数据血缘关系。通过解析SAS脚本文件，自动识别源数据字段与目标输出字段之间的映射关系，生成结构化的血缘对照表。

### 1.2 项目目标
- **核心功能**：从SAS代码中提取字段级别的血缘关系，建立源字段到目标字段的映射
- **处理范围**：支持几十个SAS文件（每个约200行）的批量分析
- **输出格式**：生成CSV格式的血缘对照表，包含以下6列信息（与参考文件Output_sas_20260318.csv完全一致）：
  1. output_table：目标表名或文件名
  2. output_column：目标字段名
  3. original_table：源表名或文件名（多源时用分号分隔）
  4. original_column：源字段名（多源时用分号分隔）
  5. formula：字段转换逻辑（passthrough表示直接传递，rename表示重命名，其他为计算表达式）
  6. script_file：处理该字段的SAS脚本文件路径
- **处理策略**：记录所有字段的完整血缘关系，包括直接传递、重命名和计算字段，并在formula列中详细记录转换逻辑

### 1.3 关键约束
- **字段名标准化**：以实际使用的字段名为准（如"EstimatedSalary"），忽略定义文件中的拼写错误
- **数据源类型支持**：必须正确识别和区分mainframe数据集、CSV文件、Excel文件、SAS临时文件（.sas7bdat）四种数据源类型
- **独立工具**：无需与其他系统集成，可独立运行
- **性能要求**：针对小规模数据集（几十个文件，每文件200行左右）进行优化
- **输出限制**：仅需血缘对照表，无需图形化展示或其他报告格式

## 2. 系统架构设计

### 2.1 整体架构
系统采用模块化设计，包含四个核心组件：

```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  SAS代码解析器   │───▶│  血缘分析引擎    │───▶│  结果输出模块   │    │  LLM增强模块    │
└─────────────────┘    └──────────────────┘    └─────────────────┘    └─────────────────┘
        │                       │                       │                       ▲
        ▼                       ▼                       ▼                       │
   SAS文件输入           血缘关系图谱           CSV血缘对照表           可选LLM调用
```

LLM增强模块为可选组件，仅在配置启用时处理规则解析无法确定的复杂场景。

### 2.2 架构特点
- **混合解析策略**：以规则解析为主（处理90%+的常规SAS语法），支持可选的LLM增强模式（处理复杂或模糊场景）
- **可配置LLM集成**：通过配置文件启用/禁用LLM Provider（如OpenAI、Anthropic等），使用langchain-ai框架实现
- **批处理**：支持一次性处理多个SAS文件，建立完整的血缘关系网络
- **内存友好**：针对小规模数据优化，避免不必要的内存开销
- **可扩展**：模块化设计便于未来功能扩展

## 3. 模块设计

### 3.1 SAS代码解析器 (SASParser)

#### 功能职责
- 解析SAS脚本文件，识别关键语法结构（包括宏变量引用）
- 提取数据集定义、字段重命名、数据集合并等操作信息
- 构建中间表示（Intermediate Representation, IR）
- 处理SAS宏变量（%let语句）并在解析过程中进行变量替换

#### 核心解析规则
1. **数据集定义识别**：
   - `DATA dataset_name;` 语句识别目标数据集
   - `SET source_dataset;` 语句识别源数据集
   - **Mainframe数据集定义**：识别 `infile "&source_path/dataset_name&file_sfx"` 和 `input` 语句块，提取字段定义
   
2. **字段重命名解析**：
   - 识别 `RENAME=(old_name = new_name)` 语法
   - 处理多字段重命名：`RENAME=(field1=new1 field2=new2)`
   
3. **字段选择解析**：
   - 识别 `KEEP field1 field2;` 语句中的字段列表
   - 识别 `DROP field1 field2;` 语句中的排除字段
   
4. **数据集合并解析**：
   - 识别 `MERGE dataset1 dataset2;` 语句
   - 识别 `BY merge_key;` 合并键
   
5. **宏变量处理**：
   - 支持解析包含宏变量的数据集名称（如 `&lib..dataset_name`）
   - 在解析阶段展开宏变量引用，确保数据集名称的正确识别

#### 输出格式
```python
{
    "file_path": "path/to/file.sas",
    "operations": [
        {
            "type": "dataset_definition",
            "target_dataset": "churn_out",
            "source_datasets": ["churn_raw"],
            "source_type": "mainframe"  // 或 "csv", "excel", "sas7bdat"
        },
        {
            "type": "rename",
            "mappings": {"CreditScore": "credit_scr", "EstimatedSalary": "est_sal"}
        },
        {
            "type": "keep",
            "fields": ["CustomerId", "Geography", "Gender", "Age", "Tenure", "Balance", "Exited", "credit_scr", "est_sal", "num_prod", "bal_per_prod", "senior_flag"]
        }
    ]
}
```

### 3.2 血缘分析引擎 (LineageAnalyzer)

#### 功能职责
- 基于解析器输出构建完整的血缘关系图谱
- 处理跨文件的数据依赖关系
- 应用字段名标准化规则（修正拼写错误）
- 生成最终的血缘映射关系

#### 核心算法
1. **血缘传播算法**：
   - 从输出数据集开始反向追踪
   - 对于重命名字段，建立 `new_field -> old_field` 映射
   - 对于直接使用的字段，建立 `field -> field` 映射
   - **多源合并处理**：当遇到MERGE操作时，分别追踪每个源数据集的字段血缘
     - **多源字段表示**：在original_table和original_column列中使用分号分隔多个源（如"churn_raw; loan_raw"和"Age; Loan_Amount"）
     - 为每个字段标记其所有原始数据源，确保血缘关系的完整性
   
2. **数据源类型识别**：
   - 通过解析上下文识别数据源类型：
     - Mainframe数据集：包含`infile`语句且路径指向mainframe系统
     - CSV文件：通过`PROC IMPORT DBMS=CSV`或`.csv`扩展名识别
     - Excel文件：通过`PROC IMPORT DBMS=EXCEL`识别
     - SAS临时文件：`.sas7bdat`扩展名
   - **计算字段处理**：记录所有字段包括计算字段，在formula列中详细记录转换逻辑：
     - 直接传递字段：formula = "passthrough"
     - 重命名字段：formula = "rename old_name -> new_name"  
     - 计算字段：formula = 完整的计算表达式（如 "(Age >= 60)" 或 "credit_scr - (dti * 10)"）

2. **跨文件依赖解析**：
   - 维护全局数据集注册表
   - 当遇到未知数据集时，在其他文件中查找定义
   - 建立完整的端到端血缘链路

3. **字段名标准化**：
   - 维护字段名修正映射表：`{"EstimagteSalary": "EstimatedSalary"}`
   - 在血缘分析过程中自动应用修正

#### 数据结构
```python
class LineageNode:
    def __init__(self, dataset_name, field_name, source_info=None):
        self.dataset_name = dataset_name
        self.field_name = field_name
        self.source_info = source_info  # (source_dataset, source_field)
        self.is_computed = False  # 标记是否为计算字段
```

### 3.3 结果输出模块 (OutputFormatter)

#### 功能职责
- 将血缘分析结果格式化为标准CSV格式
- 支持自定义输出路径和文件名
- 提供基本的验证和错误检查

#### 输出格式
CSV文件包含以下六列（与参考文件Output_sas_20260318.csv完全一致）：
- `output_table`: 目标表名或文件名
- `output_column`: 目标字段名
- `original_table`: 源表名或文件名（多源时用分号分隔）
- `original_column`: 源字段名（多源时用分号分隔）  
- `formula`: 字段转换逻辑
  - "passthrough"：直接传递字段
  - "rename old_name -> new_name"：重命名字段
  - 完整计算表达式：计算字段（如 "(Age >= 60)" 或 "credit_scr - (dti * 10)"）
- `script_file`: 处理该字段的SAS脚本文件路径

示例输出：

## 4. 数据流设计

### 4.1 处理流程
1. **文件发现阶段**：扫描指定目录，收集所有 `.sas` 文件
2. **解析阶段**：逐个解析SAS文件，生成中间表示
3. **依赖解析阶段**：构建全局数据集依赖图
4. **血缘分析阶段**：从输出数据集反向追踪到源数据集
5. **标准化阶段**：应用字段名修正规则
6. **输出阶段**：生成CSV血缘对照表

### 4.2 数据流图
```
SAS Files ──▶ Parser ──▶ IR Collection ──▶ Dependency Graph ──▶ Lineage Tracing ──▶ Standardization ──▶ CSV Output
             │          │                  │                    │                    │
             ▼          ▼                  ▼                    ▼                    ▼
         Syntax Tree  Operations      Dataset Registry    Field Mappings     Corrected Names   Final Mapping
```

## 5. 技术选型和实现方案

### 5.1 技术栈
- **编程语言**：Python 3.12
- **核心库**：
  - `re`：正则表达式解析SAS语法
  - `csv`：CSV文件读写
  - `pathlib`：文件路径处理
  - `logging`：日志记录
- **可选依赖**：
  - `langchain-ai`：用于LLM集成（可配置启用/禁用）
  - 支持多种LLM Provider（OpenAI、Anthropic等）
- **默认模式**：规则解析为主，LLM增强为可选

### 5.2 关键实现细节

#### SAS语法解析策略
采用基于正则表达式的规则解析，而非完整的语法分析器，原因：
- SAS代码结构相对简单且规律性强
- 项目规模小（几十个文件），不需要复杂的解析器
- 开发效率高，维护成本低

关键正则表达式模式：
```python
# 数据集定义
DATA_PATTERN = r'DATA\s+(\w+);'
SET_PATTERN = r'SET\s+(\w+)'
RENAME_PATTERN = r'RENAME=\(([^)]+)\)'
KEEP_PATTERN = r'KEEP\s+([^;]+);'
MERGE_PATTERN = r'MERGE\s+([^;]+);'
```

#### 字段名标准化实现
```python
FIELD_CORRECTIONS = {
    "EstimagteSalary": "EstimatedSalary",
    # 可扩展其他已知的拼写错误
}

def standardize_field_name(field_name):
    return FIELD_CORRECTIONS.get(field_name, field_name)
```

#### 血缘追踪算法
```python
def trace_lineage(target_dataset, target_field, dataset_registry):
    """反向追踪字段血缘"""
    if target_dataset not in dataset_registry:
        return None
    
    dataset_info = dataset_registry[target_dataset]
    
    # 检查是否为重命名字段
    if target_field in dataset_info.renamed_fields:
        original_field = dataset_info.renamed_fields[target_field]
        source_dataset = dataset_info.source_datasets[0]  # 简化假设单源
        return (source_dataset, original_field)
    
    # 检查是否为直接传递字段
    elif target_field in dataset_info.kept_fields:
        source_dataset = dataset_info.source_datasets[0]
        return (source_dataset, target_field)
    
    # 计算字段，跳过
    else:
        return None
```

## 6. 性能考虑

### 6.1 性能目标
- **处理能力**：支持50-100个SAS文件的批量处理
- **响应时间**：单次完整分析在10秒内完成
- **内存占用**：峰值内存使用不超过100MB

### 6.2 优化策略
1. **单次遍历**：每个SAS文件只解析一次，避免重复读取
2. **内存优化**：使用生成器和流式处理，避免全量加载
3. **缓存机制**：缓存已解析的数据集信息，避免重复解析
4. **早期终止**：对于计算字段直接跳过，不进行深度追踪

### 6.3 复杂度分析
- **时间复杂度**：O(N × M)，其中N为文件数量，M为平均文件行数
- **空间复杂度**：O(N × K)，其中K为平均字段数量

## 7. 错误处理和日志记录

### 7.1 错误处理策略
- **文件级别错误**：单个文件解析失败不影响整体处理，记录警告并继续
- **语法错误**：无法识别的SAS语法记录详细错误信息
- **依赖缺失**：找不到源数据集时记录警告，标记为外部依赖

### 7.2 日志级别设计
- **INFO**：处理进度、文件数量、成功完成
- **WARNING**：字段名修正、未知数据集、跳过的计算字段
- **ERROR**：文件读取失败、严重语法错误

### 7.3 异常处理
```python
try:
    parsed_data = parse_sas_file(file_path)
    lineage_results.extend(analyze_lineage(parsed_data))
except SASParseError as e:
    logger.warning(f"Failed to parse {file_path}: {e}")
    continue
except Exception as e:
    logger.error(f"Unexpected error processing {file_path}: {e}")
    raise
```
