# SAS数据血缘分析Python应用模块概要设计文档

## 1. 项目概述

### 1.1 背景与目标
本项目旨在开发一个独立的Python应用模块，用于分析SAS代码和数据集，自动生成数据血缘分析对照表。通过逆向追踪从输出表到源表字段的完整映射关系，帮助数据治理团队理解数据流转路径、识别数据依赖关系，并支持数据质量监控和影响分析。

### 1.2 核心需求
- **输入**：SAS代码文件（.sas）和mainframe SAS数据集定义文件
- **输出**：CSV格式的数据血缘对照表，包含源表名/字段、目标表名/字段的对应关系
- **分析方向**：从输出表作为入口，逆向追踪到源表字段
- **处理场景**：字段重命名、派生字段计算、多源数据合并等复杂转换逻辑

### 1.3 约束条件
- **规模**：处理几十个SAS代码文件，每个文件涉及几千条记录
- **性能**：无特殊性能要求，可接受合理处理时间
- **部署**：独立应用，无需集成到现有系统
- **技术栈**：Python 3.12，支持可配置的LLM Provider

## 2. 架构设计

### 2.1 整体架构
采用分层架构设计，包含以下核心组件：

```
┌─────────────────────────────────────────────────────────┐
│                    用户接口层                           │
│  - 命令行接口 (CLI)                                    │
│  - 配置管理                                            │
└─────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────┐
│                    核心处理层                           │
│  - SAS解析器：解析SAS语法结构                          │
│  - 血缘追踪器：构建数据血缘图谱                        │
│  - LLM代理：处理复杂逻辑推理                           │
│  - 结果生成器：输出标准化血缘报告                      │
└─────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────┐
│                    数据存储层                           │

└─────────────────────────────────────────────────────────┘
```

### 2.2 技术选型
- **编程语言**：Python 3.12
- **LLM框架**：LangChain + Deep Agents SDK（支持多Agent协作）
- **解析工具**：自定义SAS语法解析器（基于正则表达式和AST分析）
- **数据处理**：pandas（用于结果格式化和输出）
- **配置管理**：Pydantic（类型安全的配置模型）

### 2.3 核心设计理念
1. **增量处理**：避免一次性加载所有SAS文件到LLM上下文，采用分文件处理策略
2. **混合分析**：结合规则引擎（处理简单映射）和LLM（处理复杂逻辑）
3. **可配置性**：LLM Provider可配置，支持不同模型提供商
4. **可扩展性**：模块化设计，便于未来扩展新的SAS语法支持

## 3. 模块设计

### 3.1 SAS解析器模块 (SASParser)
**职责**：解析SAS代码文件，提取关键语法元素
- **输入**：SAS代码文件路径
- **输出**：结构化的SAS程序表示（JSON格式）

**核心功能**：
- 识别`PROC IMPORT`语句，提取源数据文件信息（包括CSV/Excel文件）
- 解析`DATA`步骤，识别输入数据集和输出数据集
- 提取`RENAME`操作，记录字段重命名映射
- 识别`KEEP`/`DROP`语句，确定字段选择范围
- 解析赋值语句，识别派生字段计算逻辑
- 处理`MERGE`/`JOIN`操作，识别多源数据合并
- **mainframe数据集定义文件解析**：解析`infile`和`input`语句，提取字段名、字段类型、字段位置等元数据信息
- **注释代码识别**：识别并标记注释掉的代码块（如注释掉的`PROC IMPORT`语句），避免误解析

**实现策略**：
- 使用正则表达式匹配SAS关键字和语法结构
- 构建抽象语法树（AST）表示程序逻辑
- 维护符号表跟踪变量作用域和数据流
- 对于mainframe数据集定义文件，专门解析固定宽度格式的`input`语句，提取字段位置偏移量和数据类型

### 3.2 血缘追踪器模块 (LineageTracker)
**职责**：基于解析结果构建完整的数据血缘图谱
- **输入**：SAS解析器输出的结构化数据
- **输出**：血缘关系图（有向图结构）

**核心功能**：
- **前向追踪**：从源表到目标表的数据流向
- **逆向追踪**：从目标表到源表的血缘路径（主要需求）
- **多源合并处理**：处理来自多个源表的字段组合
- **派生字段分析**：解析复杂表达式中的源字段依赖

**多源合并处理算法**：
- 解析`MERGE`语句中的`IN=a`、`IN=b`等标记，确定每个输入表的标识符
- 对于同名字段，根据`MERGE`语句的上下文和`IN`标记确定字段来源表
- 在输出血缘关系中正确标记多个源表和字段，支持一对多的血缘映射
- 处理`UPDATE`、`MODIFY`等其他多源操作的特殊情况

**算法设计**：
```python
# 伪代码示例
def build_lineage_graph(parsed_sas_files):
    graph = DirectedGraph()
    for file in parsed_sas_files:
        # 处理简单字段映射（passthrough）
        for field_mapping in file.field_mappings:
            if field_mapping.type == "passthrough":
                graph.add_edge(
                    source=f"{field_mapping.source_table}.{field_mapping.source_field}",
                    target=f"{field_mapping.target_table}.{field_mapping.target_field}",
                    formula="passthrough"
                )
        
        # 处理派生字段（需要LLM分析）
        for derived_field in file.derived_fields:
            source_fields = llm_agent.analyze_formula(derived_field.formula)
            for source_field in source_fields:
                graph.add_edge(
                    source=source_field,
                    target=f"{derived_field.table}.{derived_field.field}",
                    formula=derived_field.formula
                )
    
    return graph
```

### 3.3 LLM代理模块 (LLMAgent)
**职责**：处理复杂的逻辑推理任务，特别是派生字段的源字段识别
- **输入**：SAS派生字段表达式、上下文信息
- **输出**：源字段列表和转换公式说明

**核心功能**：
- **公式解析**：分析SAS表达式，识别参与计算的源字段
- **上下文理解**：结合SAS程序上下文理解字段含义
- **多Agent协作**：使用Deep Agents SDK实现专家分工

**Agent设计**：
- **FormulaAnalyzerAgent**：专门分析数学表达式和逻辑运算
- **ContextResolverAgent**：解析字段在程序上下文中的真实来源
- **ValidatorAgent**：验证血缘关系的合理性

**提示工程**：
```python
PROMPT_TEMPLATE = """
```

### 3.5 数据集定义管理模块 (DatasetDefinitionManager)
**职责**：管理mainframe数据集定义文件，提供字段元数据查询服务
- **输入**：数据集定义文件目录（def_*.sas）
- **输出**：数据集字段元数据和映射关系

**核心功能**：
- 扫描和识别数据集定义文件（def_*.sas）
- 解析数据集定义，提取字段元数据（字段名、类型、位置偏移量等）
- 维护数据集名称到定义文件的映射关系
- 提供数据集字段查询接口，支持血缘追踪时的字段信息获取

**数据集名称匹配规则**：
采用方案A：解析def_*.sas文件中的`data`语句，提取实际的数据集名称。具体规则如下：
- 查找`data &lib.. dataset_name`格式的语句（如`data &lib.. churn_raw`）
- 提取`dataset_name`部分作为数据集名称（如`churn_raw`）
- 建立数据集名称到定义文件的映射关系，用于后续字段元数据查询

**实现策略**：
- 使用SAS解析器模块解析def_*.sas文件中的`infile`和`input`语句
- 构建内存中的数据集定义缓存，提高查询性能
- 支持数据集别名映射，处理SAS代码中的不同引用方式

### 3.6 文件依赖分析模块 (FileDependencyAnalyzer)
**职责**：分析SAS文件间的依赖关系，确定处理顺序
- **输入**：所有SAS代码文件和中间文件（.sas7bdat）
- **输出**：文件依赖关系图和处理顺序

**核心功能**：
- 识别中间文件（.sas7bdat）作为输入和输出
- 构建文件间的依赖关系图（哪个文件的输出是哪个文件的输入）
- 确定文件处理顺序（拓扑排序），确保依赖文件先被处理
- 处理循环依赖检测，提供错误报告
- **中间文件字段结构获取**：从生成该中间文件的SAS代码的`DATA`步骤、`KEEP`语句和赋值语句中推断字段列表，在文件依赖分析时将中间文件的字段信息一并提取和缓存，供后续血缘追踪使用

**实现策略**：
- 解析每个SAS文件的输入输出声明（SET、MERGE、PROC EXPORT等）
- 构建有向无环图（DAG）表示文件依赖关系
- 使用拓扑排序算法确定最优处理顺序
- 对于循环依赖，提供详细的依赖路径报告

### 3.4 结果生成器模块 (ResultGenerator)
**职责**：将血缘图谱转换为标准化的CSV输出格式
- **输入**：血缘关系图
- **输出**：符合要求的CSV文件

**输出格式**：
| 字段名 | 说明 |
|--------|------|
| output_table | 目标表名 |
| output_column | 目标字段名 |
| original_table | 源表名（多源时用分号分隔） |
| original_column | 源字段名（多字段时用分号分隔） |
| formula | 转换公式或操作类型 |
| script_file | SAS脚本文件路径 |

## 4. 数据流设计

### 4.1 整体处理流程
```
1. 配置加载 → 2. SAS文件发现 → 3. 分文件解析 → 4. 血缘构建 → 5. 结果输出
     ↑              ↓                   ↓               ↓             ↓
     └────── LLM配置 ←─────── LLM代理调用 ←───── 复杂逻辑处理 ←───── 格式化
```

### 4.2 详细处理步骤

#### 步骤1：初始化与配置
- 加载应用配置（LLM Provider、输入目录、输出路径等）
- 初始化LLM客户端和Agent系统

#### 步骤1.5：数据集定义加载
- 扫描数据集定义文件（def_*.sas）
- 解析并缓存数据集字段定义
- 构建数据集名称到定义的映射表

#### 步骤2：SAS文件发现与预处理
- 扫描指定目录，收集所有.sas文件
- 过滤注释和无关内容，准备解析

#### 步骤3：分文件解析
- 使用SAS解析器逐个解析SAS文件
- **关联数据集定义**：将SAS代码中的数据集名称与定义文件关联
- 对于未找到定义的数据集，记录警告
- 对每个SAS文件独立调用SAS解析器
- 生成结构化的程序表示
- 缓存解析结果到内存

#### 步骤4：血缘关系构建
- **简单映射处理**：直接构建passthrough和rename关系
- **复杂逻辑处理**：对派生字段调用LLM代理分析
- **多源合并处理**：识别来自不同源表的字段组合
- 构建完整的血缘有向图

#### 步骤5：结果生成与输出
- 遍历血缘图，按输出表分组
- 生成标准化CSV记录
- 写入输出文件

### 4.3 血缘分析算法

#### 逆向追踪算法
```python
def trace_lineage_backward(target_table, target_field, lineage_graph):
    """从目标字段逆向追踪到所有源字段"""
    visited = set()
    result = []
    
    def dfs(current_node):
        if current_node in visited:
            return
        visited.add(current_node)
        
        # 获取当前节点的所有上游依赖
        upstream_edges = lineage_graph.get_incoming_edges(current_node)
        if not upstream_edges:
            # 叶子节点，是源字段
            result.append(current_node)
        else:
            # 递归处理上游节点
            for edge in upstream_edges:
                dfs(edge.source)
    
    dfs(f"{target_table}.{target_field}")
    return result
```

#### 多源字段处理
对于来自多个源表的字段（如merged_out中的CustomerId），算法需要：
1. 识别MERGE/JOIN操作中的多个输入表
2. 确定字段在各输入表中的存在性
3. 在输出中正确标记多个源表和字段

## 5. 接口设计

### 5.1 配置接口
使用Pydantic定义配置模型：

```python
from pydantic import BaseModel
from typing import Optional

class LLMConfig(BaseModel):
    provider: str  # "openai", "anthropic", "ollama"等
    model: str
    api_key: Optional[str] = None

class AppConfig(BaseModel):
    input_directory: str
    output_file: str
    llm_config: LLMConfig
    dataset_definition_directory: str  # 数据集定义文件目录
    dataset_definition_pattern: str = "def_*.sas"  # 定义文件匹配模式
    base_url: Optional[str] = None
    max_concurrent_files: int = 5  # 控制并发处理文件数
```

### 5.2 命令行接口
```bash
# 基本用法
python sas_lineage.py --input-dir ./sas_files --output-file lineage.csv

# 指定LLM配置
python sas_lineage.py --input-dir ./sas_files --output-file lineage.csv \
    --llm-provider openai --llm-model gpt-4o --api-key YOUR_KEY

# 使用配置文件
python sas_lineage.py --config config.yaml
```

### 5.3 输入输出接口

#### 输入要求
- **SAS代码文件**：.sas扩展名，包含完整的数据处理逻辑
- **目录结构**：所有相关SAS文件放在同一目录或子目录中
- **文件编码**：UTF-8编码

#### 输出格式
严格遵循示例Output_sas_20260318.csv的六列格式：
1. `output_table`：目标表名
2. `output_column`：目标字段名  
3. `original_table`：源表名（多源用分号分隔）
4. `original_column`：源字段名（多字段用分号分隔）
5. `formula`：转换公式或操作类型（passthrough/rename/表达式）
6. `script_file`：生成该映射的SAS脚本文件路径

## 8. 风险与缓解措施

### 8.1 主要风险
1. **LLM理解偏差**：复杂SAS逻辑可能被LLM误解
2. **SAS语法覆盖不全**：未支持的SAS语法导致解析失败
3. **多源合并复杂性**：复杂的JOIN逻辑难以准确追踪
4. **数据集定义文件缺失或格式错误**：mainframe数据集定义文件缺失或格式不符合预期

### 8.2 缓解措施
1. **混合验证**：结合规则引擎和LLM，互相验证结果
2. **人工审核模式**：提供交互式审核界面确认可疑结果
3. **渐进式支持**：优先支持常见SAS语法，逐步扩展
4. **测试用例覆盖**：建立完整的测试用例库验证准确性
5. **默认字段推断机制**：对于缺失定义的数据集，提供默认字段推断机制，并记录警告日志
