## 1. 项目概述

### 1.1 项目背景
本项目旨在开发一个Python应用模块，用于分析SAS代码和相关数据集，自动生成数据血缘分析对照表。该系统将帮助数据工程师和分析师理解数据从源系统到最终输出的完整流转路径，特别是在复杂的SAS数据处理环境中。

### 1.2 核心需求
- **输入数据类型**：
  - SAS代码文件（.sas）
  - 源数据（主要为mainframe定义，部分为CSV/Excel文件）
  - 中间临时文件（.sas7bdat格式）
  - 输出文件（CSV格式）

- **输出要求**：
  - 数据血缘对照表，包含：源表名/文件名、源表字段、目标表名/文件名、目标表字段
  - 从输出表作为分析入口，反向追踪每个结果字段对应的源字段
  - 支持复杂的字段转换逻辑识别（重命名、计算表达式、多源合并等）

- **技术约束**：
  - 处理大量SAS代码文件时需考虑上下文长度限制
  - 需要可配置的LLM Provider支持
  - 基于Python 3.12开发
  - 利用langchain-ai和deep agents SDK构建智能代理系统

### 1.3 示例场景分析
基于提供的示例文件，我们识别出以下典型的数据血缘模式：

1. **简单字段映射**：源字段直接传递到目标字段（passthrough）
2. **字段重命名**：源字段通过RENAME操作映射到新字段名
3. **派生字段**：通过计算表达式创建新字段（如 `bal_per_prod = Balance / num_prod`）
4. **多源合并**：来自不同源表的字段在目标表中组合（如 `risk_score = credit_scr - (dti * 10)`）
5. **条件逻辑**：基于条件表达式创建标志字段（如 `senior_flag = (Age >= 60)`）

### 2.2 代理角色分工

#### 2.2.1 Master Orchestrator Agent（主编排代理）
- **职责**：整体任务协调、进度管理、结果整合
- **核心能力**：
  - 使用`write_todos`工具进行任务分解和进度跟踪
  - 动态调度子代理执行特定任务
  - 处理异常情况和重试逻辑
  - 最终结果验证和格式化输出

#### 2.2.2 Code Analysis Agent（代码分析代理）
- **职责**：解析SAS代码，提取数据处理逻辑
- **核心能力**：
  - 识别数据输入/输出语句（PROC IMPORT/EXPORT, DATA steps）
  - 提取字段映射关系和转换表达式
  - 识别数据集依赖关系（SET, MERGE操作）
  - 处理SAS宏变量和条件逻辑

#### 2.2.3 Data Mapping Agent（数据映射代理）
- **职责**：建立源数据到目标数据的完整映射关系
- **核心能力**：
  - 解析mainframe数据定义文件（def_*.sas）
  - 处理CSV/Excel源数据的字段信息
  - 构建中间数据集的字段血缘图谱
  - 处理字段重命名和类型转换

#### 2.2.4 Lineage Validation Agent（血缘验证代理）
- **职责**：验证和优化生成的血缘关系
- **核心能力**：
  - 一致性检查（确保所有字段都有明确的源）
  - 循环依赖检测
  - 血缘路径完整性验证
  - 生成最终标准化的输出格式

### 2.3 数据流设计
1. **输入阶段**：用户指定SAS代码目录和输出文件列表
2. **预处理阶段**：文件系统扫描，识别所有相关文件（.sas, .csv, .sas7bdat）
3. **分析阶段**：
   - 代码分析代理处理每个SAS文件
   - 数据映射代理解析源数据定义
   - 构建局部血缘关系图
4. **整合阶段**：主编排代理合并所有局部关系，构建全局血缘图
5. **验证阶段**：血缘验证代理检查结果完整性
6. **输出阶段**：生成标准化的血缘对照表

## 4. 数据血缘分析算法

### 4.1 多阶段分析策略
为应对大量SAS代码文件的上下文限制问题，采用分阶段分析策略：

#### 阶段0：数据定义预处理
- 扫描所有def_*.sas文件，提取mainframe数据集的字段定义
- 建立数据集名称到字段模式的映射表
- 为后续血缘分析提供源数据字段的元信息

#### 阶段1：文件级预处理
- 将大型SAS文件按逻辑块分割（DATA steps, PROC statements）
- 为每个代码块生成摘要和关键信息提取
- 建立文件间的依赖关系图

#### 阶段2：局部血缘分析
- 对每个代码块独立分析，生成局部血缘关系
- 使用向量存储保存中间结果，支持后续检索
  - **存储内容**：每个SAS代码块的摘要、输入输出关系、字段映射
  - **检索场景**：当分析某个代码块需要引用其他代码块的上下文时
  - **检索方式**：基于代码块路径或数据集名称进行相似性检索
- 记录不确定或需要上下文的分析点

#### 阶段3：全局血缘整合
- 基于文件依赖关系，合并局部血缘图
- 处理跨文件的字段引用和数据流
- 解决冲突和不一致的映射关系

#### 4.2.1 字段映射识别算法
```python
def identify_field_mappings(sas_code: str) -> List[FieldMapping]:
    """
    识别SAS代码中的字段映射关系
    
    映射类型包括：
    1. 直接传递 (passthrough): field1 -> field1
    2. 重命名 (rename): old_name -> new_name  
    3. 派生字段 (derived): expr -> new_field
    4. 条件字段 (conditional): condition -> flag_field
    """
    mappings = []
    
    # 步骤1: 识别RENAME操作
    rename_mappings = extract_rename_operations(sas_code)
    mappings.extend(rename_mappings)
    
    # 步骤2: 识别赋值表达式
    assignment_mappings = extract_assignment_expressions(sas_code)
    mappings.extend(assignment_mappings)
    
    # 步骤3: 识别KEEP/DROP语句中的直接传递
    passthrough_mappings = extract_passthrough_fields(sas_code)
    mappings.extend(passthrough_mappings)
    
    return mappings
```

#### 4.2.2 反向血缘追踪算法
```python
def trace_reverse_lineage(output_field: str, output_table: str, 
                         lineage_graph: LineageGraph) -> List[SourceField]:
    """
    从输出字段反向追踪到源字段
    
    算法步骤：
    1. 在血缘图中定位输出字段节点
    2. 执行深度优先搜索，找到所有源字段
    3. 记录完整的转换路径和表达式
    4. 处理多源合并的情况
    """
    source_fields = []
    visited = set()
    
    def dfs(current_field, path):
        if current_field in visited:
            return
        visited.add(current_field)
        
        # 如果是源字段（没有入边），添加到结果
        if lineage_graph.in_degree(current_field) == 0:
            source_fields.append(SourceField(
                field=current_field,
                path=path.copy(),
                expression=get_full_expression(path)
            ))
            return
        
        # 递归处理所有上游字段
        for predecessor in lineage_graph.predecessors(current_field):
            path.append(predecessor)
            dfs(predecessor, path)
            path.pop()
    
    dfs(FieldNode(output_table, output_field), [])
    return source_fields
```

### 4.3 处理复杂场景的策略

#### 4.3.1 多源合并处理
对于MERGE操作产生的字段，算法需要：
- 识别合并键（BY variables）
- 分别追踪来自不同源表的字段
- 在输出中明确标注多源字段的来源
  - **输出格式**：original_table和original_column字段使用分号分隔多个源值
  - **合并键处理**：合并键字段在输出中保持单一源表引用（通常是第一个源表）

#### 4.3.2 宏变量和动态代码处理
- 静态分析无法确定的动态代码部分，标记为需要人工验证
- 提供上下文信息帮助LLM理解宏变量的实际值
- 建立宏变量到实际值的映射表

#### 4.3.3 循环和条件逻辑处理
- 识别DO循环和IF-THEN-ELSE结构
- 为条件分支创建独立的血缘路径
- 在输出中标注条件逻辑的影响范围

```python
        )
    
    def index_analysis_results(self, results: List[AnalysisResult]):
        """索引分析结果到向量存储"""
        documents = []
        for result in results:
            doc = Document(
                page_content=str(result),
                metadata={"file": result.source_file, "type": "lineage"}
            )
            documents.append(doc)
        
        splits = self.text_splitter.split_documents(documents)
        self.vector_store = InMemoryVectorStore.from_documents(
            splits, self.embeddings
        )
    
    def retrieve_relevant_context(self, query: str, k: int = 5):
        """检索相关的血缘上下文"""
        if not self.vector_store:
            return []
        return self.vector_store.similarity_search(query, k=k)
```

### 7.5 异常处理

系统需要处理以下异常情况：

- **SAS代码语法错误**：记录错误日志，跳过该文件，继续处理其他文件
- **文件不存在**：记录警告，标记该数据集为外部依赖
- **LLM调用失败**：重试机制（最多3次），失败后标记为需要人工验证
- **字段映射无法确定**：输出中标记为"UNKNOWN"，记录待验证列表
