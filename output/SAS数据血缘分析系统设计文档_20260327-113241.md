# SAS数据血缘分析系统技术设计文档

## 1. 项目概述

### 1.1 项目背景
本项目旨在开发一个Python应用模块，用于分析SAS代码和数据集，生成数据血缘分析对照表。系统需要处理复杂的SAS数据处理流程，包括从mainframe数据源、CSV/Excel文件到中间临时文件（.sas7bdat）再到最终输出CSV文件的完整数据流转过程。

### 1.2 核心需求
- **反向血缘追踪**：从输出表作为分析入口，找到每个结果表字段对应的源表字段
- **多源数据支持**：处理mainframe定义的数据集、CSV/Excel文件、中间.sas7bdat文件
- **大规模处理**：支持大量SAS代码和文件的分析，避免一次性提交给LLM导致上下文溢出
- **血缘关系类型**：识别直通字段、重命名字段、计算字段、多源组合字段等血缘关系

### 1.3 输出格式
生成标准的数据血缘对照表，包含以下字段：
- 源表名/文件名
- 源表字段  
- 目标表名/文件名
- 目标表字段
- 转换公式（可选）
- 脚本文件路径（可选）

### 3.2 SAS解析模块（SASParserAgent）
**职责**：解析SAS代码，提取数据处理逻辑和字段映射关系
- **输入**：SAS脚本文件路径
- **输出**：结构化的处理步骤和字段转换规则

#### 文件类型识别规则
- **Transform脚本**：包含`PROC EXPORT`、`DATA`步骤中包含`SET`/`MERGE`语句的文件
- **Definition脚本**：包含`infile`和`input`语句但无`PROC EXPORT`的文件（mainframe数据集定义）

#### Mainframe数据集定义文件解析规则
针对def_Churn.sas、def_train.sas等mainframe定义文件：

1. **`infile`语句解析**：
   - 提取文件路径：`"&source_path/churn_raw&file_sfx"` → 解析为实际路径或保留宏变量
   - 提取记录格式：`recfm=F`（固定长度记录格式）
   - 提取记录长度：`lrecl=657`（记录长度657字节）

2. **`input`语句解析**：
   - 字段起始位置：`@1`, `@3`, `@5`等表示字段在记录中的字节偏移位置
   - 字段名称：支持带引号的字段名如`'Loan Amount'`
   - 格式说明符识别：
     - `PKn.m`：压缩十进制格式（Packed Decimal），n为总位数，m为小数位
     - `S370FPDn.m`：IBM S/370浮点十进制格式
     - `$EBCDICn`：EBCDIC字符编码，n为字符长度

3. **字段元数据提取**：
   ```python
   # 伪代码示例
   def parse_input_statement(input_lines):
       fields = []
       for line in input_lines:
           if '@' in line:
               position = extract_position(line)  # 提取@后的数字
               field_name = extract_field_name(line)  # 提取字段名
               format_spec = extract_format(line)  # 提取格式说明符
               data_type = infer_data_type(format_spec)  # 推断数据类型
               fields.append({
                   'name': field_name,
                   'position': position,
                   'format': format_spec,
                   'data_type': data_type
               })
       return fields
   ```

#### Transform脚本SAS语法解析规则

1. **DATA步骤解析**：
   - **SET语句**：识别源数据集引用
     ```sas
     data output;
         set source1 source2;  /* 多个源数据集 */
     ```
   - **MERGE语句**：识别多表连接
     ```sas
     data merged;
         merge table1(in=a) table2(in=b);
         by key_field;
     ```
   - **KEEP/DROP语句**：字段选择/排除
     ```sas
     keep field1 field2;  /* 只保留指定字段 */
     drop field3 field4;  /* 排除指定字段 */
     ```
   - **RENAME语句**：字段重命名
     ```sas
     rename old_name = new_name;
     ```
   - **计算字段**：表达式提取
     ```sas
     new_field = field1 + field2 * 0.1;
     ```

2. **PROC步骤解析**：
   - **PROC IMPORT**：外部文件导入
     ```sas
     proc import datafile="input.csv" out=dataset;
     ```
   - **PROC EXPORT**：结果导出（标识输出文件）
     ```sas
     proc export data=final_result outfile="output.csv";
     ```
   - **PROC SORT**：排序操作（不影响血缘关系）

3. **宏变量处理**：
   - 识别宏变量引用：`&lib..table_name`, `"&source_path/file"`
   - 宏变量展开：根据上下文或配置替换为实际值

4. **条件逻辑处理**：
   - IF-THEN-ELSE语句中的字段赋值
   - DO-END块中的复杂逻辑
   - WHERE语句中的过滤条件（不影响血缘，但需记录）

#### 解析算法伪代码

1. **主解析流程**：
```python
def parse_sas_file(file_path: str) -> ParsedSASFile:
    """主解析函数"""
    content = read_file(file_path)
    statements = tokenize_sas_content(content)
    
    if is_definition_script(statements):
        return parse_definition_script(statements, file_path)
    else:
        return parse_transform_script(statements, file_path)

def is_definition_script(statements: List[str]) -> bool:
    """判断是否为mainframe定义脚本"""
    has_infile = any('infile' in stmt.lower() for stmt in statements)
    has_input = any('input' in stmt.lower() for stmt in statements)
    has_export = any('proc export' in stmt.lower() for stmt in statements)
    return has_infile and has_input and not has_export
```

2. **Transform脚本解析**：
```python
def parse_transform_script(statements: List[str], file_path: str) -> ParsedSASFile:
    """解析transform脚本"""
    data_steps = extract_data_steps(statements)
    proc_steps = extract_proc_steps(statements)
    
    transformations = []
    output_files = []
    
    for data_step in data_steps:
        # 解析DATA步骤
        output_dataset = extract_output_dataset(data_step)
        sources = extract_sources(data_step)  # SET/MERGE语句
        operations = extract_operations(data_step)  # KEEP/DROP/RENAME/计算
        
        transformation = TransformationRule(
            output=output_dataset,
            sources=sources,
            operations=operations,
            source_file=file_path
        )
        transformations.append(transformation)
    
    for proc_step in proc_steps:
        if proc_step.type == 'EXPORT':
            output_files.append(proc_step.outfile)
    
    return ParsedSASFile(
        file_type='transform',
        transformations=transformations,
        output_files=output_files
    )
```

#### 智能体协作机制设计

##### SupervisorAgent调度算法
SupervisorAgent使用分层任务分解策略：
1. **第一层分解**：按文件类型分解
   - Definition脚本解析任务
   - Transform脚本解析任务  
   - 输出文件验证任务

2. **第二层分解**：按依赖关系分解
   - 基于拓扑排序确定处理顺序
   - 识别可并行处理的独立子图
   - 为每个子图分配独立的分析会话

3. **第三层分解**：按文件大小分解
   - 大文件（>1MB）分块处理
   - 小文件批量处理（减少LLM调用开销）
   - 动态调整分块大小基于可用内存

##### Agent通信协议
所有智能体通过标准化消息格式进行通信：

```python
class AgentMessage:
    """智能体间通信消息格式"""
    def __init__(self, sender: str, receiver: str, message_type: str, payload: dict):
        self.sender = sender                  # 发送者Agent名称
        self.receiver = receiver              # 接收者Agent名称  
        self.message_type = message_type      # 消息类型：task/request/response/error
        self.payload = payload                # 消息载荷
        self.timestamp = time.time()          # 时间戳
        self.session_id = None                # 会话ID（用于跟踪）
```

消息类型定义：
- `task`：任务分配消息（Supervisor → Worker）
- `request`：数据请求消息（Worker → Worker）
- `response`：数据响应消息（Worker → Requester）
- `error`：错误报告消息（任何Agent → Supervisor）
- `progress`：进度更新消息（Worker → Supervisor）

##### 上下文窗口管理
- **动态分配策略**：根据任务复杂度分配上下文窗口
  - 简单任务：500 tokens
  - 中等任务：1000 tokens  
  - 复杂任务：2000 tokens（最大值）

- **上下文压缩**：当接近上限时自动压缩上下文
  - 移除已完成任务的历史记录
  - 摘要化中间结果
  - 保留关键决策点和错误信息

- **会话隔离**：每个分析任务使用独立的会话上下文
  - 避免不同任务间的上下文污染
  - 支持并行任务处理
  - 便于错误隔离和调试

## 4. 数据处理流程

### 4.1 整体处理流程
```
1. 初始化 → 2. 文件发现 → 3. 依赖分析 → 4. 任务规划 → 
5. 并行解析 → 6. 血缘构建 → 7. 结果验证 → 8. 报告生成
```

### 4.2 详细步骤说明

#### 步骤1：初始化
- 加载系统配置（LLM Provider、文件路径、输出格式等）
- 初始化文件系统Backend和检查点存储
- 创建唯一的分析会话ID

#### 步骤2：文件发现
- 扫描指定目录，识别所有SAS相关文件
- 分类文件类型：transform脚本、definition脚本、输出CSV、中间文件
- 构建初始文件依赖图

#### 步骤3：依赖分析
- 分析SAS脚本间的依赖关系（输出文件作为其他脚本的输入）
- 识别处理流水线的拓扑排序
- 确定分析入口点（最终输出表）

#### 步骤4：任务规划
- 基于依赖图，将分析任务分解为独立的子任务
- 为每个子任务分配适当的上下文窗口大小
- 制定执行顺序和并发策略

#### 步骤5：并行解析
- 启动多个SASParserAgent实例，并行处理不同脚本
- 每个解析任务在独立的智能体会话中执行
- 解析结果暂存到中间缓存

#### 步骤6：血缘构建
- LineageAnalyzerAgent从输出表开始反向追踪
- 递归解析每个字段的源依赖
- 构建完整的血缘图谱

#### 步骤7：结果验证
- 验证血缘关系的完整性（无孤立节点）
- 检测潜在的数据质量问题（空值传播、类型不匹配等）
- 生成验证报告

#### 步骤8：报告生成
- 格式化血缘关系为标准对照表
- 支持多种输出格式（CSV、JSON、可视化图谱）
- 保存分析元数据（时间戳、版本信息等）

### 4.3 异常处理

#### SAS代码解析失败处理
- **语法错误**：当SAS代码包含不支持的语法或格式错误时：
  - 记录详细的错误信息（文件路径、行号、错误描述）
  - 尝试跳过错误部分，继续解析其他有效代码
  - 在最终报告中标记受影响的字段和表
  - 提供错误摘要供用户手动修复

- **不支持的语法**：对于系统未实现的SAS语句：
  - 维护已知不支持语法的白名单
  - 对于白名单中的语法，记录警告但继续处理
  - 对于未知语法，抛出异常并停止当前文件解析

#### 循环依赖检测和处理
- **检测机制**：在构建血缘图谱时实时检测循环依赖
  ```python
  def detect_cycles(graph: nx.DiGraph) -> List[List[str]]:
      """检测图中的循环依赖"""
      return list(nx.simple_cycles(graph))
  ```
- **处理策略**：
  - 发现循环依赖时，记录循环路径
  - 标记受影响的字段为"循环依赖"
  - 提供循环依赖的详细路径信息
  - 允许用户配置是否中断分析或继续处理

#### 缺失源文件处理
- **文件不存在**：当引用的源文件（mainframe定义、中间文件等）不存在时：
  - 记录缺失文件的引用路径
  - 创建虚拟节点表示缺失的源字段
  - 在血缘关系中标记"源文件缺失"
  - 继续分析其他可解析的部分

- **部分文件缺失**：当只有部分源文件缺失时：
  - 优先处理完整的依赖链
  - 对不完整的依赖链进行标记
  - 提供缺失文件列表供用户补充

#### LLM调用失败处理
- **API错误**：LLM Provider返回错误时：
  - 实现指数退避重试机制（最多3次重试）
  - 记录失败的请求和响应
  - 切换到备用LLM Provider（如果配置了多个）
  - 对于持续失败的情况，使用规则引擎降级处理

- **超时处理**：
  - 为每个LLM调用设置合理的超时时间（默认60秒）
  - 超时后自动重试或使用缓存结果
  - 记录超时统计用于性能优化

#### 系统级异常处理
- **内存不足**：大文件处理时内存不足：
  - 动态调整分块大小
  - 启用磁盘缓存替代内存缓存
  - 释放不必要的中间结果

- **并发限制**：达到系统并发限制时：
  - 使用任务队列管理待处理任务
  - 动态调整并发度基于系统负载
  - 提供进度反馈避免用户误以为系统卡死

#### 异常日志和监控
- **结构化日志**：所有异常都记录结构化日志，包含：
  - 时间戳、会话ID、文件路径、错误类型、错误详情
  - 上下文信息（当前处理状态、输入参数等）
  
- **监控指标**：
  - 解析成功率/失败率
  - 异常类型分布
  - 平均重试次数
  - 处理时间分布

## 5. 血缘分析算法

### 5.1 血缘图谱数据结构

### 5.3 反向追踪算法
```python
def reverse_trace_lineage(target_field, lineage_graph):
    """从目标字段反向追踪到源字段"""
    visited = set()
    path = []
    
    def dfs(current_field):
        if current_field in visited:
            return
        visited.add(current_field)
        path.append(current_field)
        
        # 获取当前字段的所有上游依赖
        predecessors = lineage_graph.predecessors(current_field)
        for pred in predecessors:
            dfs(pred)
    
    dfs(target_field)
    return path
```

```python
class LineageNode:
    """血缘图谱中的节点（表字段）"""
    def __init__(self, table_name: str, field_name: str, data_type: str = None):
        self.table_name = table_name          # 表名/文件名
        self.field_name = field_name          # 字段名
        self.data_type = data_type            # 数据类型（从mainframe格式推断）
        self.source_file = None               # 源文件路径
        self.source_line = None               # 源代码行号
        self.is_source_field = False          # 是否为源字段（mainframe定义）
        self.is_output_field = False          # 是否为输出字段（CSV列）
        
    @property
    def node_id(self) -> str:
        """节点唯一标识符：table_name.field_name"""
        return f"{self.table_name}.{self.field_name}"
```

```python
class LineageEdge:
    """血缘图谱中的边（转换关系）"""
    def __init__(self, source_node: LineageNode, target_node: LineageNode):
        self.source = source_node             # 源节点
        self.target = target_node             # 目标节点
        self.transformation_type = "passthrough"  # 转换类型：passthrough/rename/computed/merged

        self.source_line = None               # 转换发生的代码行号
```

```python
class FieldDefinition:
    """mainframe字段定义"""
    def __init__(self, name: str, position: int, format_spec: str):
        self.name = name                      # 字段名称
        self.position = position              # 字节偏移位置
        self.format_spec = format_spec        # 格式说明符（PK2.0, S370FPD6.0, $EBCDIC3等）
        self.data_type = self._infer_data_type()  # 推断的数据类型
        
    def _infer_data_type(self) -> str:
        """根据格式说明符推断数据类型"""
        if self.format_spec.startswith('PK') or self.format_spec.startswith('S370FPD'):
            return 'numeric'
        elif self.format_spec.startswith('$EBCDIC'):
            return 'character'
        else:
            return 'unknown'
```

#### 血缘图谱数据结构
```python
import networkx as nx

class LineageGraph:
    """血缘图谱主类"""
    def __init__(self):
        self.graph = nx.DiGraph()             # 有向图存储血缘关系
        self.nodes = {}                       # 节点字典：node_id -> LineageNode
        self.edges = []                       # 边列表
        
    def add_node(self, node: LineageNode):
        """添加节点到图谱"""
        self.graph.add_node(node.node_id)
        self.nodes[node.node_id] = node
        
    def add_edge(self, edge: LineageEdge):
        """添加边到图谱"""
        self.graph.add_edge(edge.source.node_id, edge.target.node_id)
        self.edges.append(edge)
        
    def get_predecessors(self, node_id: str) -> List[str]:
        """获取指定节点的所有前驱节点"""
        return list(self.graph.predecessors(node_id))
        
    def get_successors(self, node_id: str) -> List[str]:
        """获取指定节点的所有后继节点"""
        return list(self.graph.successors(node_id))
```
